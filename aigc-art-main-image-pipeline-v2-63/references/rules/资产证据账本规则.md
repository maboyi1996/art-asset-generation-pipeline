# 资产证据账本规则

## 规则目的

本文件只用于约束 `asset-evidence-ledger.md` 的生成方式。

本规则解决的唯一问题是：

> 在候选准入和 S/A/B/C 评级之前，如何从场次中心事实表转成资产类型中心的三张无损证据账本？

本规则不替代 `资产候选准入规则.md`、三类评级规则、`状态拆分规则.md` 或人物候选去向规则。它只生成评级前证据账本，不决定候选是否进入资产池、不写污染清洗台账、不评级、不注册、不生成任务。

## 输入依据

- `scene-coverage-table`
- 原剧本 `source_evidence` / `source_locator`

不得绕过 `script-intake-result` 重新切场。允许回看原文定位来核验证据主体、可见性、组合关系、同一性线索和状态线索归属，但不得在本层重写其他层判断。

## 输出结果

输出结果固定命名为：

- `character-evidence-ledger`
- `scene-evidence-ledger`
- `prop-evidence-ledger`

三张账本都是评级和候选准入的证据输入，不是候选去向台账，不是生产任务表，不计入最终任务数量。

## 共同记录原则

- 一条资产证据一行；同一场内有多个明确主体时应拆成多行。
- `raw_label` 或对应 raw 字段保留原文或覆盖记录中的表述；归一字段只写可被可靠识别的干净主体。
- 证据必须能回溯到 `episode_id`、`scene_ids`、`source_locator` 和完整语义 `source_evidence`。
- 证据文本不得只截取有歧义的孤立短语；必须保留足够上下文来判断“提起”等多义动作、台词对象、VO/OS、闪回/梦境、纪念物指代。
- 同一人物、同一空间、同一物件证据不足时，只记录线索和不确定性，不在本层合并或升级。
- 状态线索只记录归属和证据，不生成状态资产；状态采用、排除或送审由 `state-variant.md` 决定。

## 人物证据账本字段

`character-evidence-ledger` 必填字段：

| 字段 | 规则 |
|---|---|
| `evidence_id` | 稳定证据 ID。 |
| `asset_type` | 固定为 `character`。 |
| `raw_label` | 原文人物称呼、台词前缀或覆盖记录标签。动作、情绪、语气、声音、组合文本可保留在本字段或原文证据。 |
| `normalized_subject` | 能可靠识别主体时填写规范主体；不能识别时留空并写明原因。 |
| `presence_type` | 继承或核验为 `physical_visible`、`memory_visible`、`dream_visible`、`portrait_visible`、`corpse_visible`、`voice_only`、`mentioned_only`、`memorial_reference`、`rumor_mentioned` 等。 |
| `episode_id` | 来源集号。 |
| `scene_ids` | 来源场次 ID。 |
| `source_locator` | 页码、行号、文本块或可回溯定位。 |
| `source_evidence` | 支持该条证据的完整语义原文。 |
| `visibility_for_rating` | `visible_face_evidence`、`non_visible_evidence`、`evidence_only`。 |
| `label_disposition` | `clean_subject`、`normalized_from_raw_label`、`split_from_combination`、`generic_label`、`transferable_title`、`unresolved_label`。 |
| `same_person_evidence_note` | 同一人物证据、别名线索、稳定关系、连续事件链或证据不足说明。 |
| `state_clue_note` | 属于该人物的可见状态线索；无明确主体时不得挂到人物。 |
| `ledger_uncertainty_note` | 证据冲突、主体不明、可见性不明或需要回上游复核时必填。 |

### 人物记录范围

- 正常可见人物、闪回/梦境/想象画面中的可见人物。
- 画像、尸体、遗体等需要人物形象生产的可见媒介。
- VO、OS、对白提及、纪念物指代等非真人可见证据；这些必须记录，但不得作为跨集真人脸模升级证据。
- 泛称人物、群体人物、背景功能人物。
- 台词前缀、动作/情绪/语气/声音修饰、组合短语、未消歧可转移称谓。

### 原文标签与主体归一示例

本层允许无损保留原文标签，但必须把“原文标签”和“可被后续候选准入读取的主体”拆开：

| 原文/覆盖记录 | `raw_label` | `normalized_subject` | `label_disposition` | 本层处理 |
|---|---|---|---|---|
| `三夫人害怕` | `三夫人害怕` | `三夫人` | `normalized_from_raw_label` | `害怕` 只保留为原文证据或状态线索候选，不进入主体名。 |
| `棠宁声音沙哑` | `棠宁声音沙哑` | `宋棠宁` | `normalized_from_raw_label` | `声音沙哑` 只留证据，不进人物名或状态名。 |
| `萧厌轻声` | `萧厌轻声` | `萧厌` | `normalized_from_raw_label` | `轻声` 只留证据，不进人物名或状态名。 |
| `二夫人三夫人一起` | `二夫人三夫人一起` | `二夫人`、`三夫人` 分别成行 | `split_from_combination` | 拆成两个主体证据；不得出现组合主体。 |
| `宋父温柔：宁宁乖……` | `宋父温柔` | `宋父` | `normalized_from_raw_label` | 宋父通常按上下文判断为说话主体；宁宁是否在场不能自动判定。 |

本层不决定以上记录是否进入 `polluted-label-cleanup-ledger`。是否记录污染标签清洗，由 `asset-candidate-admission.md` 在候选准入时按 `污染标签清洗规则.md` 决定。

如果原文标签无法可靠识别主体，`normalized_subject` 必须留空，`label_disposition` 写 `unresolved_label`，并在 `ledger_uncertainty_note` 指明需要回 `scene-coverage.md` 或人工复核。不得为了不断链把整句污染短语写成主体。

## 场景证据账本字段

`scene-evidence-ledger` 必填字段：

| 字段 | 规则 |
|---|---|
| `evidence_id` | 稳定证据 ID。 |
| `asset_type` | 固定为 `scene`。 |
| `raw_location_text` | 原文地点或空间表述。 |
| `normalized_scene_subject` | 便于候选承接的空间主体；证据不足时不要过度合并。 |
| `space_scope` | `specific_space`、`parent_location`、`generic_space`、`transition_space`、`mentioned_only`。 |
| `episode_id` | 来源集号。 |
| `scene_ids` | 来源场次 ID。 |
| `source_locator` | 页码、行号、文本块或可回溯定位。 |
| `source_evidence` | 支持该条证据的完整语义原文。 |
| `same_space_evidence_note` | 同一空间证据、归属者、房间功能、独特布置、上下场承接或证据不足说明。 |
| `visual_design_clues` | 可见空间设计线索，如牌匾、布局、材质、天气、仪式布置、破损、火灾等。 |
| `state_clue_note` | 属于该空间的可见状态线索。 |
| `ledger_uncertainty_note` | 空间层级、泛化地点、同一空间证据不足或冲突时必填。 |

### 场景记录范围

- 明确地点、房间、院落、宫殿、街巷、牢房、审堂等空间。
- 泛化地点和过渡空间，例如门口、走廊、街上、屋内。
- 只在对白中提及但可能影响场景判断的地点。
- 与状态相关的空间线索，例如火灾、夜晚、封闭、宴席、祭祀、审判布置。

## 道具证据账本字段

`prop-evidence-ledger` 必填字段：

| 字段 | 规则 |
|---|---|
| `evidence_id` | 稳定证据 ID。 |
| `asset_type` | 固定为 `prop`。 |
| `raw_prop_text` | 原文物件或道具表述。 |
| `normalized_prop_subject` | 便于候选承接的道具主体；证据不足时不要过度合并。 |
| `prop_scope` | `specific_item`、`tracked_item`、`generic_item`、`background_set_dressing`、`mentioned_only`。 |
| `episode_id` | 来源集号。 |
| `scene_ids` | 来源场次 ID。 |
| `source_locator` | 页码、行号、文本块或可回溯定位。 |
| `source_evidence` | 支持该条证据的完整语义原文。 |
| `interaction_type` | `visible_present`、`used`、`close_up`、`exchanged`、`opened`、`damaged`、`hidden`、`evidence_object`、`mentioned_only` 等。 |
| `same_item_evidence_note` | 同一物件证据、交接链、独特标记、所有权、状态变化或证据不足说明。 |
| `visual_state_clues` | 染血、破损、封存、打开、虫卵、铭文、材质等可见物件状态。 |
| `ledger_uncertainty_note` | 同一物件证据不足、泛化物件、普通陈设或冲突时必填。 |

### 道具记录范围

- 实体出现、被使用、被交换、被打开、被损坏、被藏匿或被特写的物件。
- 承担证据、身份、信物、机关、毒物、武器、文书功能的物件。
- 普通陈设和背景物件；可进入账本但不得因频次自动升级。
- 只在对白、心理或旁白中提及的物件；必须标明 `mentioned_only`。

## 交接要求

- `asset-candidate-admission.md` 必须从三张账本读取证据并生成 `asset-candidate-list`、污染清洗记录、可见排除或阻断记录。
- `asset-rating.md` 主要读取三张账本进行 S/A/B/C 评级，并可回看 `source_locator` 对应原文核验。
- `state-variant.md` 读取三张账本中的状态线索归属和可见差异证据，但不得把主体不明线索静默生成状态。
- 评级层发现本账本漏掉关键可见事实时，不得自行静默补齐；必须输出阻断并指回 `asset-evidence-ledger.md` 或 `scene-coverage.md`。

## 边界

- 不做候选准入。
- 不输出 `polluted-label-cleanup-ledger`。
- 不评级 S/A/B/C。
- 不拆分状态资产。
- 不判断 `task_type`。
- 不设置 `review_status`。
- 不生成候选最终去向。
- 不注册资产。
- 不生成任务、提示词或分集映射。
