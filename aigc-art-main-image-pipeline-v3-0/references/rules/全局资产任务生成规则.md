# 全局资产任务生成规则

## 规则目的

本文件只用于约束 `task-manifest-generation.md` 如何把已注册、已拆状态或已确定人物去向的资产映射为 `global-task-draft`。

本规则解决的唯一问题是：

> 哪些角色、场景、道具 base asset 或 state asset 必须形成可被中台消费的生产任务行，哪些只能形成可见排除记录？

本规则补齐 `global-task-draft` 的生成口径，防止 B/C 人物因为不需要项目级脸模锚点而只有人物去向、没有 `text_to_image` 任务和中文 core prompt。

V3.0 硬边界：全部可见 C 级角色不得合并成模板。只要 C 级候选被保留为可见人物生产资产，就必须生成独立 `text_to_image` 基础任务；不生产时必须有明确排除、非视觉或人工审核去向。

## 输入依据

必须读取前置层级已经产出的结构化结果：

- `global-asset-registry`
- `state-variant-result`
- `character-disposition-ledger`
- `generic-template-coverage-ledger`

不得跳过前置结果重新抽取资产、重新拆状态、重新评级、重新注册资产或重写候选去向。`priority_level` 从 `global-asset-registry` 继承，状态从 `state-variant-result` 继承。

`polluted-label-cleanup-ledger` 不是本层主输入；污染清洗记录不得生成生产任务，污染断链由 `production-audit.md` 检查。

## 输出结果

输出结果固定命名为 `global-task-draft`。

`global-task-draft` 是中台生产任务的草稿源。后续 `episode-usage-map.md`、`character-anchor.md`、`production-audit.md`、`prompt-generation.md`、`output-contracts.md` 都只能读取本表中已经存在的任务，不得在下游补建任务。

## 任务生成原则

每个已注册且应生产的 base asset 或 state asset 必须进入 `global-task-draft`。不生产的资产必须形成可见排除记录，或被 `character-disposition-ledger` / `generic-template-coverage-ledger` 引用，不得静默丢弃。

污染标签清洗记录不是资产注册结果，不得生成 `global-task-draft` 任务。被准入门挡下的污染标签只能进入 `polluted-label-cleanup-ledger`，不能用清洗记录替代真实人物候选或生产任务。

| 资产情况 | `global-task-draft` 处理 |
|---|---|
| S/A 人物基础形象 | 必须生成 `text_to_image` 任务，用于后续脸模锚点。 |
| S/A 人物状态图 | 必须生成状态任务；默认 `image_text_edit`，并在后续引用锚点。 |
| B 人物基础形象 | 必须生成 `text_to_image` 任务，`episode_id` 必填。 |
| B 人物状态图 | 不生成。B 级人物只生成基础人物图；状态线索不进入 `state-variant-result`。 |
| C 可视单人人物 | 保留为资产时必须生成独立基础 `text_to_image` 任务；不生产时必须有可见排除原因；不生成状态图。 |
| C 泛称、群体或功能人物 | 保留为可见生产资产时必须生成独立基础 `text_to_image` 任务；不生产时必须有分组、可见排除、非视觉或人工审核原因；不生成状态图；不得合并成模板任务。 |
| C 非视觉或只提及人物 | 不生成生产任务，但必须进入人物候选去向台账并写明 `excluded_reason` 或 `not_visual_candidate` 原因。 |
| S/A/B/C 场景 | 需要生产主图时生成 `text_to_image` 任务；不生产时必须有可见排除说明。 |
| S/A/B/C 道具 | 需要生产主图时生成 `text_to_image` 任务；不生产时必须有可见排除说明。 |
| S/A 场景或道具状态图 | 有 `state-variant-result` 状态时生成状态任务；通常为 `text_to_image`，除非任务类型规则允许编辑型任务。 |

## 候选去向联动

- `character-disposition-ledger.final_disposition` 为 `production_task` 时，`global-task-draft` 必须生成同 ID `task_id`。
- `generic-template-coverage-ledger` 不得生成可见 C 级人物模板生产任务；覆盖表只用于分组、排除、非视觉或人工审核说明。
- `state-variant-result` 中需要生产的状态资产必须生成任务行，且通过 `base_asset_id` 继承 `priority_level`。
- B/C 资产不得拥有状态任务；即使原文存在服装、伤病、脏污、血迹、空间或物体状态线索，也不得由本层补建状态任务。
- `manual_review_required` 只能作为任务行的 `review_status`，不得替代 `task_id` 或最终生产去向。
- `grouped_into` 必须指向真实集合、别名目标、场面覆盖或可见排除记录，不得指向空分组或替代可见 C 级独立任务。
- B/C 可生产人物不得只有 `character-disposition-ledger` 记录而没有对应生产任务。

## 任务字段要求

生成任务行必须符合 `任务类型字段审核规则.md` 的字段和枚举要求。

| 字段 | 规则 |
|---|---|
| `task_id` | 生成行必填，稳定可追踪；生产类候选去向已有计划 ID 时必须沿用该 ID。 |
| `base_asset_id` | 必填；继承 `global-asset-registry.base_asset_id`，不得临时拼接或只用资产名代替。 |
| `state_asset_id` | 状态任务必填并继承 `state-variant-result.state_asset_id`；基础任务为空。 |
| `asset_type` | 只使用 `character`、`scene`、`prop`。 |
| `asset_name` | 使用注册后的规范资产名。 |
| `state_name` | 状态任务填写状态名；基础任务写默认状态。 |
| `priority_level` | 继承 `global-asset-registry`，状态继承 base asset 等级；不得在本规则重新评级。 |
| `same_asset_evidence` | 继承同一资产证据。 |
| `task_type` | 只允许 `text_to_image` 或 `image_text_edit`。 |
| `anchor_task_id` | 仅 `image_text_edit` 必填；`text_to_image` 为空。 |
| `review_status` | 只允许 `ready`、`manual_review_required`、`excluded`。 |
| `review_reason` | 存疑或排除时必填。 |
| `episode_id` | B 级单集资产必填；其他资产按分集映射规则处理。 |
| `source_evidence` | 必须引用剧本事实或场次证据。 |
| `prompt`、`edit_instruction`、`negative_prompt` | 本层不生成正文，字段由 `prompt-generation.md` 补齐。 |
| `review_focus` | 本层可写审核关注点，供后续提示词和人审读取。 |

## 泛称和群体人物任务约束

泛称、群体或背景人物不得因为在多集或多场重复出现，就自动生成多个独立单人脸模任务；但一旦被保留为可见 C 级生产资产，就不得合并成模板任务，必须独立生成 `text_to_image` 任务。

- 没有同一具体人物证据的 `下人`、`侍卫`、`丫鬟`、`小厮`、`黑衣人`、`太监`、`百姓` 等，如无独立可见生产需求，默认走 `grouped_into`、`excluded` 或 `not_visual_candidate`。
- 只有当该泛称、群体或背景人物被明确保留为可见生产资产时，才生成 C 级独立 `text_to_image` 任务。
- 同一泛称标签被拆成多个任务时，必须能说明每个任务的场面差异、造型差异或可见生产需求差异，而不是仅因 episode_id 不同。
- 不得使用泛称模板任务合并可见 C 级角色。

## 可见排除记录

只有以下情况允许不生成生产任务：

- 候选只被提及，没有可见形象生产需求。
- 候选是纯功能性背景，不需要独立人物、场景或道具资产。
- 候选已由明确场面覆盖、分组或可见排除记录覆盖。
- 剧本证据冲突导致无法生产，但必须生成 `review_status: manual_review_required` 的任务行或阻断审计问题。

排除记录必须写明 `review_reason`，或在人物候选去向台账中写明 `excluded_reason`，并能被 `production-audit.md` 检查。

## 边界

- 不重新抽取资产。
- 不重新拆状态。
- 不重新评级 S/A/B/C。
- 不注册资产。
- 不改人物候选去向。
- 不读取污染清洗台账来生成任务。
- 不建立人物脸模锚点。
- 不生成 prompt、edit instruction 或 negative prompt 正文。
- 不定义最终 Markdown 排版。
- 不新增中台生产任务字段。
- 不为 JSON 投影临时补造 `base_asset_id` 或 `state_asset_id`。
- 不修改 `task_type` 或 `review_status` 枚举。

只使用以上列举过的字段、枚举和职责。
