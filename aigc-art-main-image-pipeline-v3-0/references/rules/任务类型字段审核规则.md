# 任务类型字段审核规则

## 规则目的

本文件只定义机器长表需要暴露给中台的生产字段、任务类型枚举、审核状态枚举和计数口径。

本文件的核心问题是：

> 一条进入中台的生产任务行，必须有哪些字段？这些字段允许哪些取值？

本文件不定义资产评级、状态拆分、角色锚定规则、分集映射规则、提示词写法或最终 Markdown 排版。

哪些已评级资产必须生成任务，由 `全局资产任务生成规则.md` 定义。本文件只审核已经进入全局生产任务表的行是否合法。

## 允许的任务类型

中台生产任务类型只有两种：

| `task_type` | 含义 | 适用对象 | 必要输入 |
|---|---|---|---|
| `text_to_image` | 纯文生图。 | 人物主图、B 级单集连续人物、可见 C 级独立人物、场景图、道具图。 | 中文 core prompt、中文负向约束、证据 |
| `image_text_edit` | 锚定图 + 文生图。 | 用于 S/A 人物状态图；B/C 不生成状态图。 | `anchor_task_id`、参考图、中文状态编辑 core instruction、中文负向约束 |

`task_type` 只使用上表列举的两种取值。

分集复用通过 `分集映射规则.md` 指向同一个 `task_id` 表达，不另造任务类型。

## 审核状态

审核状态与任务类型分离。存疑任务仍然是正常任务行，只是 `review_status: manual_review_required`。

| `review_status` | 含义 | 是否需要提示词 | 是否进入一键生成 |
|---|---|---:|---:|
| `ready` | skill 判断明确。 | 生成行需要 | 是 |
| `manual_review_required` | skill 产出可用任务，但需要人工确认后再生成。 | 生成行需要 | 人工确认后进入 |
| `excluded` | 不生成该资产行。 | 不需要 | 否 |

所有 `manual_review_required` 和 `excluded` 行必须填写 `review_reason`。

## 机器长表必要字段

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `task_id` | 生成行必填 | 稳定任务 ID |
| `base_asset_id` | 是 | 必须回指 `global-asset-registry.base_asset_id`，用于 JSON 元素聚合 |
| `state_asset_id` | 条件必填 | 状态任务必填并回指 `state-variant-result.state_asset_id`；基础任务为空 |
| `asset_type` | 是 | `character`、`scene`、`prop` |
| `asset_name` | 是 | 规范化资产名称 |
| `state_name` | 有状态时必填 | 状态名称；无拆分时写默认状态 |
| `priority_level` | 是 | `S/A/B/C`；状态继承 base asset 等级 |
| `same_asset_evidence` | 是 | `strong/medium/weak/conflict` |
| `task_type` | 生成行必填 | `text_to_image` 或 `image_text_edit` |
| `anchor_task_id` | 条件必填 | `image_text_edit` 必填；`text_to_image` 为空 |
| `review_status` | 是 | `ready/manual_review_required/excluded` |
| `review_reason` | 条件必填 | 存疑或排除时必填 |
| `episode_id` | 条件必填 | B 级单集连续资产必填；S/A 可在分集映射体现 |
| `source_evidence` | 是 | 简短剧本证据 |
| `prompt` | 条件必填 | `text_to_image` 生成行必填；内容是中文 core prompt，等待中台拼接风格前缀 |
| `edit_instruction` | 条件必填 | `image_text_edit` 必填；内容是中文状态编辑 core instruction |
| `negative_prompt` | 生成行必填 | 中文负向约束 |
| `review_focus` | 是 | 人工审核重点 |

机器长表主字段只使用上表列举字段；新增字段必须先更新本规则。

`base_asset_id` 和 `state_asset_id` 是 JSON 投影的稳定连接字段，不得删除、改名或仅用 `asset_name/state_name` 替代。无法回指注册表或状态索引表时，必须由审计或终检阻断。

本文件只检查提示词相关字段是否存在，不评价提示词内容质量。提示词内容由提示词模板和守卫约束。

V3.0 的 `prompt`、`edit_instruction`、`negative_prompt` 必须是中文。`prompt` 和 `edit_instruction` 不得包含题材世界段、画法段、`{{include:...}}`、风格前缀或旧英文写实模板残留。

## 评级前证据账本字段边界

`character-evidence-ledger`、`scene-evidence-ledger`、`prop-evidence-ledger` 和机器长表中的三类评级前证据账本不属于中台生产任务行。

- 三类 evidence ledger 不生成 `task_id`。
- 三类 evidence ledger 不使用 `task_type`。
- 三类 evidence ledger 不使用 `review_status`。
- 三类 evidence ledger 不参与 `production_task_count`、`ready_task_count`、`manual_review_task_count` 或 `excluded_count`。
- `source_evidence_ids` 可出现在候选、注册、人物去向、污染清洗和对应机器长表审计章节中，用于回指 evidence ledger。
- `rating_evidence_ids` 可出现在评级结果、注册表和对应机器长表审计章节中，用于回指评级依据。
- `source_evidence_ids` 和 `rating_evidence_ids` 不得加入全局生产任务表主字段，除非另行修改中台生产字段契约。
- `base_asset_id` 和 `state_asset_id` 是生产任务到注册表、状态索引表和 JSON manifest 的连接字段，不属于 evidence ledger 字段。

## 计数口径

项目概览需要拆开统计：

| 计数字段 | 定义 |
|---|---|
| `production_task_count` | `task_type` 为 `text_to_image/image_text_edit` 且 `review_status` 不是 `excluded` 的行 |
| `ready_task_count` | `review_status: ready` 的行 |
| `manual_review_task_count` | `review_status: manual_review_required` 的行 |
| `excluded_count` | `review_status: excluded` 的行 |
| `episode_mapping_row_count` | 分集映射行数，不等于生产任务数 |

生产任务计数只统计机器长表中的生成行；注册表行、状态索引行、候选去向行、污染清洗行、泛称/群体模板覆盖行和分集映射行单独统计。

## 人物候选去向台账边界

`character-disposition-ledger` 和机器长表中的 `人物候选去向审计表` 不属于中台生产任务行。

- `final_disposition` 不是 `task_type`，不得替代或扩展 `text_to_image/image_text_edit`。
- `人物候选去向审计表` 不参与 `production_task_count`、`ready_task_count`、`manual_review_task_count` 或 `excluded_count`。
- 只有当 `final_disposition` 指向 `production_task` 时，才通过 `task_id` 关联到全局生产任务表。V3.0 中可见 C 级人物不得通过 `template_task` 合并进入生产。
- 不得把 `candidate_id`、`final_disposition`、`alias_to`、`grouped_into`、`excluded_reason` 等台账字段加入全局生产任务表主字段。
- `manual_review_required` 是任务行的 `review_status`，不是 `final_disposition` 的生产替代品。

## 污染标签清洗记录边界

`polluted-label-cleanup-ledger` 和机器长表中的 `污染标签清洗记录表` 不属于中台生产任务行。

- 不生成 `task_id`。
- 不使用 `task_type`。
- 不使用 `review_status`。
- 不参与人物候选数、S/A/B/C 数、生产任务数、`ready_task_count`、`manual_review_task_count` 或 `excluded_count`。
- 不得把清洗记录字段加入全局生产任务表字段。

## 泛称/群体模板覆盖边界

`generic-template-coverage-ledger` 和机器长表中的 `泛称/群体模板覆盖表` 不属于中台生产任务行。

- `coverage_result` 不是 `task_type`。
- V3.0 中泛称/群体覆盖表不得用来合并可见 C 级人物生产任务。非视觉或无生产需求的泛称/群体可以分组、排除或标记非视觉；被保留为可见 C 级生产资产时必须进入独立 `production_task`。
- 本表不参与 `production_task_count`、`ready_task_count`、`manual_review_task_count` 或 `excluded_count`。
- 不得把 `coverage_id`、`coverage_result`、`generic_label`、`coverage_evidence` 等字段加入全局生产任务表字段。

## 状态线索去向边界

`state-clue-disposition-ledger` 和机器长表中的 `状态线索去向表` 不属于中台生产任务行。

- `disposition` 不是 `task_type`、`review_status` 或人物 `final_disposition`。
- 只有被采用或复用为状态资产的线索，才通过 `state_asset_id` 关联到状态索引和后续任务。
- 本表只允许 S/A 状态识别范围内的数据；B/C、泛称、群体、普通背景资产或无法指向 S/A 资产的状态线索不得出现。
- 本表不参与生产任务计数。
- 不得把 `clue_id`、`raw_clue`、`disposition`、`exclusion_reason` 等字段加入全局生产任务表字段。

## 边界

- 不判断资产是否成立。
- 不判断 `priority_level`。
- 不拆分状态。
- 不建立人物锚点。
- 不判断分集复用。
- 不生成提示词正文。
- 不生成 JSON。

只使用以上列举过的字段和职责。
