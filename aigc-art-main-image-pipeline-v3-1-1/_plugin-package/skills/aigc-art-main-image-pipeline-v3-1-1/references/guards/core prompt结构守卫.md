# core prompt 结构守卫

## 目的

检查人物、场景、道具主图提示词是否是等待中台拼接风格前缀的中文 core prompt。

本守卫只检查结构和边界，不自动补写提示词，不选择画风，不拼接风格前缀。

## 通用必查

- 提示词必须是中文 core prompt。
- 不得包含 `{{include:...}}`。
- 不得包含 `【A. 题材世界】` 或 `【B. 画法】`。
- 不得包含风格库前缀全文。
- 不得残留旧版英文写实模板片段，例如 `real portrait photograph`、`photographic realism`、`raw portrait photography`、`realistic pores`、`PBR asset look`。
- 必须包含图像类型、画幅比例、主体身份、可见状态、构图或版式要求、负向约束。
- 不得用剧情摘要替代可见画面描述。
- 不得用抽象评价替代可画细节，例如只写“相符、一致、自然、端正、稳定、清楚、有辨识度、符合阶层属性、由称谓确定”。

## 人物

人物 core prompt 必须包含：

- 单人胸像或半身主设定图要求。
- 中性背景要求。
- 角色身份。
- 脸部特征。
- 发型或修饰。
- 身形或体态。
- 服装或可见装饰。
- 眼神、表情或情绪核心。
- 脸、发、服装、眼神和表情必须能看到具体形状、颜色、位置、材质、状态或强度，不得只写身份气质判断。
- 防止多人物、剧情截图、动作戏和遮脸的负向约束。

## 场景

场景 core prompt 必须包含：

- 主场景资产图要求。
- 无人物规则。
- 准确空间身份。
- 场景定位。
- 观察点与空间关系。
- 前中后景或空间层次。
- 核心画面元素和材质状态。
- 光影、色彩或氛围。
- 空间结构、前中后景、元素材质、光源和空气状态必须具体可画，不得只写“空间布局清楚”或“氛围有辨识度”。
- 防止错误空间、人物、无关道具、可读文字和水印的负向约束。

## 道具

道具 core prompt 必须包含：

- 道具资产参考图要求。
- 单个道具主体。
- 正反面或完整结构展示要求。
- 准确道具身份。
- 归属与用途。
- 核心部件、材质、颜色和新旧状态。
- 整体状态。
- 道具必须写实体结构、部件、材质、颜色、磨损或使用痕迹，不得只写“符合身份”“状态明确”。
- 防止人物、手、环境、错误道具、无关物件、可读文字和水印的负向约束。

## 边界

- 不判断资产是否应该生产。
- 不重新抽取、评级或拆状态。
- 不改写提示词。
- 不读取风格前缀文件。
- 不把真实摄影、赛博朋克或其他画风作为默认风格写入 core prompt。
