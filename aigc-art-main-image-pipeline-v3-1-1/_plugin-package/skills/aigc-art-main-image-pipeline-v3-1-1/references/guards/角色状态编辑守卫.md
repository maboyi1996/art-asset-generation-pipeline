# 角色状态编辑守卫

## 目的

保证人物状态图基于已确认脸模生成，不把状态图写成新角色设计，并保证 `edit_instruction` 是中文 core instruction。

## 核心规则

人物状态编辑必须同时使用：

- `identity_reference_image`
- `anchor_task_id`
- `stable_face_identity`
- `state_delta_prompt`
- `edit_instruction`

`edit_instruction` 不得包含题材世界段、画法段、`{{include:...}}`、风格前缀或旧版英文写实模板残留。

## 状态差异范围

`state_delta_prompt` 只允许写可见人物外观变化：

- 服装类别、轮廓、制服、伪装、礼服、工服。
- 发型、胡须、妆容、湿发、凌乱。
- 伤病、疲惫、脏污、血迹、怀孕、年龄状态。
- 阶层或身份呈现导致的可见造型变化。

## 禁止

- 改脸、换演员、改年龄段、改脸部比例。
- 把地点、道具、动作、对白、关系、胜负、复仇等剧情事件写成状态差异。
- 用纯情绪做状态，除非情绪留下稳定可见痕迹。
- 用语气、声音、说话方式或瞬时表情做状态，例如低声、声音沙哑、眼眸温柔、惊喜、担忧、不甘。
- 在缺少明确主体证据时，把场景里的受伤、脏污、血迹、衣衫凌乱等关键词套到该人物身上。
- 使用英文旧模板句式。
- 拼接中台负责处理的画风前缀。

## 失败处理

缺少锚点图或状态证据不足时，设置 `review_status: manual_review_required` 并写明审核原因。

如果状态名或 `state_delta_prompt` 本身非法，应回到 `state-variant.md`，不得仅靠改写 `edit_instruction` 通过。
