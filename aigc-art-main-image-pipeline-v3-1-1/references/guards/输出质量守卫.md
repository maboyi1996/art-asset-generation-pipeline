# 输出质量守卫

## 目的

检查最终交付是否满足 V3.1.1 输出契约。

本守卫只检查文件、章节、字段、引用和审计状态，不重新抽取资产、不重新拆状态、不重新评级、不补任务、不改变输出。

## 必查项

- 是否输出 `artifact-manifest.json`。
- 是否输出 `audit/`、`production-source/`、`deliverables/` 三个目录。
- `artifact-manifest.json` 是否登记所有必需 audit、production source、deliverable 和 validation report。
- 是否输出 `production-source/core-prompt-slots.md`。
- 是否输出 `production-source/prompted-production-task-table.md`。
- 是否输出 `production-source/main-image-production-table.md`。
- 是否输出 `deliverables/prompt-manifest.json`。
- 是否输出 `deliverables/main-image-review-table.docx`。
- 生产源表是否包含项目概览、全局资产注册表、状态索引表、角色身份锚点表、全局生产任务表、分集使用映射表和生产审计摘要。
- 生产源表是否没有完整人物评级前证据账本、场景评级前证据账本、道具评级前证据账本、人物候选去向审计表、泛称/群体模板覆盖表、污染标签清洗记录表和状态线索去向表。
- 完整中间审计产物是否保存在 `audit/` 对应独立文件中，而不是塞入生产源表。
- DOCX 是否按集包含 `人物表`、`场景表`、`道具表`。
- DOCX 是否没有 `基础提示词` 列。
- DOCX 是否没有输出 `prompt`、`edit_instruction`、`negative_prompt`、完整 core prompt 或风格前缀。
- DOCX 人物表是否包含 `人物`、`本集状态与外貌`、`评级`、`人设图`、`5视图`。
- DOCX 图片列是否为空白占位，不包含复制图片或生成图片。
- 生产源表中的全局生产任务表字段是否只使用 `任务类型字段审核规则.md` 和 `机器长表模板.md` 允许的生产字段。
- 全局生产任务表是否包含 `base_asset_id` 和 `state_asset_id`；`base_asset_id` 必须能回指全局资产注册表，状态任务的 `state_asset_id` 必须能回指状态索引表。
- 全局资产注册表是否不包含旧版状态资产列表字段或任何状态 ID 列表；状态只能出现在状态索引表。
- 状态索引表和状态线索去向表是否只包含 S/A 状态范围内的数据；B/C、泛称、群体、普通背景资产或无法指向 S/A 资产的状态线索不得出现。
- 人物候选去向审计字段、泛称/群体模板覆盖字段、污染标签清洗记录字段、状态线索去向字段是否没有污染全局生产任务表字段。
- `production-audit-result` 是否仍有 `severity: blocking`；如有，最终交付必须标记未通过，不得在终检层修复。
- 每个 `text_to_image` 生产任务是否填写中文 `prompt` 和中文 `negative_prompt`。
- 每个 `image_text_edit` 生产任务是否填写中文 `edit_instruction`、中文 `negative_prompt` 和有效 `anchor_task_id`。
- `prompt` 和 `edit_instruction` 是否没有 `{{include:...}}`、`【A. 题材世界】`、`【B. 画法】`、风格前缀或旧英文写实模板残留。
- 每条分集映射是否引用真实 `task_id`，或通过 `disposition_ref` 指向可见排除、非视觉去向或泛称覆盖记录。
- `task_type` 是否只使用 `text_to_image` 和 `image_text_edit`。
- `review_status` 是否只使用 `ready`、`manual_review_required`、`excluded`。
- `priority_level` 是否只使用 `S/A/B/C`，状态是否继承 base asset 等级，且状态行只限 S/A。
- B/C 可生产资产是否只生成基础任务或合法排除/非视觉去向；不得存在 B/C 状态任务、B/C `image_text_edit` 状态链路或 B/C 状态人工复核记录。
- 可见 C 级人物是否拥有独立 `text_to_image` 任务；不得用模板任务、空分组或覆盖记录替代。
- 污染标签是否只出现在三类 evidence ledger 的 raw 字段、`source_evidence` 原文证据或 `污染标签清洗记录表.raw_label`。
- `polluted-label-cleanup-ledger` 是否没有被计入人物候选数、S/A/B/C 数、生产任务数或审核状态统计。
- `deliverables/prompt-manifest.json` 是否由 manifest 登记的生产源表投影生成，且没有反向减少、合并、改名、补建或重算生产源表任务。
- `deliverables/main-image-review-table.docx` 是否由 manifest 登记的生产源表派生，且没有反向减少、合并、改名、补建或重算生产源表任务。

## 非阻断项

- 不因项目集数长、B/C 总量低、模板数量少或等级比例异常直接判定最终质检失败。
- 不因 B/C 状态线索没有出现在状态索引表、状态线索去向表或状态人工复核中判定最终质检失败。
- 上述数量异常只能在生产审计或人审 DOCX 中作为 `warning`/`info` 呈现，不能替代逐事实、逐候选、逐状态、逐任务、逐映射的闭环检查。

## 失败处理

发现文件缺失、manifest 缺登记、role 错误、章节缺失、字段非法、引用断链、提示词字段缺失、污染标签进入结构化名称、production-source/DOCX/JSON 分工错误或 `production-audit-result` 仍有 blocking 时，返回对应拥有层修复，不在最终质检层重写结果。
