# 场景主图提示词模板

## 用途

供 `scripts/render_core_prompts.py` 根据 `production-source/core-prompt-slots.md` 渲染场景资产中文 core prompt。V3.1.1 中 `prompt-generation.md` 只填写槽位，不直接手写整段 prompt。

本模板不包含题材世界段、画法段或风格前缀。中台会在本 core prompt 前拼接运营选择的风格前缀。

## 核心原则

本模板原始设计有 A「题材世界」和 B「场景画法」两层；V3.1.1 改为由中台后续拼接，所以模板正文只生成 C 之后的 core prompt。D/E/F/G 只写场景身份、空间结构、元素材质、光影和氛围事实，不写题材词、画法词、渲染词或风格前缀。C 段人物规则和末尾负向约束是生产防错位置；D/E/F/G 段不得写否定、排除、纠错或对照句式。缺失元素直接省略，不声明其不存在。负向约束只负责生产防错，不承担题材或画法控制。

## 主图 core prompt 模板

```text
图像类型：
主场景资产图。

画幅比例：
横版 16:9。

人物规则：
无人物，无可识别人物，无人形主体。

画面规则：
空间布局清楚，前景、中景、背景层次明确，入口、主空间、视觉中心和空间纵深清楚；细节丰富但不杂乱，主视觉锚点明确。

场景名称：
[填写场景名称]

准确空间身份：
[具体空间类型 / 所属地点或建筑层级 / 室内外关系 / 空间功能 / 阶层属性 / 一眼识别的空间特征]

场景定位：
[城市 / 区域 / 建筑级别 / 室内外 / 楼层或空间位置 / 公共或私密 / 阶层属性 / 空间类型]

时间状态：
[白天 / 夜晚 / 雨夜 / 清晨 / 事件发生前 / 事件发生后]

剧情功能：
[这个场景承担什么剧情作用：交易、相遇、审判、逃亡、压迫、暴露秘密、关系转折、情绪爆发、隐藏信息]

一句话定位：
[精准场景定位 + 核心空间锚点 + 场景功能]

观察点与空间关系：
[镜头从哪里看向空间，入口 / 过渡 / 主空间如何成立]

视觉中心：
[画面最重要的空间锚点]

前景：
[遮挡、引导或建立空间入口的元素]

中景：
[主空间与视觉中心]

背景：
[深处结构、延展空间或暗示区域]

空间暗示：
[不完整出现但需要让人感到存在的区域]

必须出现：
- [核心元素 + 材质 + 状态 / 使用痕迹]
- [核心元素 + 材质 + 状态 / 使用痕迹]

可选细节：
- [辅助元素 / 背景细节 + 材质 + 状态 / 使用痕迹]
- [辅助元素 / 背景细节 + 材质 + 状态 / 使用痕迹]

主光源：
[填写主光源]

辅助光：
[填写辅助光]

明暗关系：
[填写明暗关系]

色彩倾向：
[填写色彩倾向]

空气状态：
[填写空气状态]

氛围与情绪：
[基础氛围 / 心理压力 / 空间温度]

负向约束：
不要错误空间身份；不要错误阶层质感；不要让空间布局不可读；不要让核心场景元素消失；不要随机堆满无关道具；不要变成动作截图；不要人物；不要可读文字、真实品牌标识或水印。
```

## 约束

- 最终提示词必须是中文 core prompt。
- 场景主图默认空场或近空场。
- 不为普通场景自动添加人群。
- 时间、光照和空间状态必须来自剧本证据或保守推断。
- D/E/F/G 段只写正向可见事实，不写否定、排除、纠错或对照句式。
- 缺失元素直接省略，不写不存在元素的否定声明。
- 防人物、防动作、防错空间、防无关道具等生产防错只写在人物规则、负向约束或 `negative_prompt`。
- 一句话定位必须是正向定位句，不写纠错对照句。
- 不写题材世界段、画法段、`{{include:...}}` 或风格库前缀。
