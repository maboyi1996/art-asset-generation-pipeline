# 资产候选准入规则

## 规则目的

本文件约束 `asset-candidate-list` 的准入边界。

本层的目的不是给资产定级，而是从三类评级前证据账本中承接人物、场景和道具候选项，同时过滤明显不是资产的内容。它负责回答“证据账本里的事实如何进入候选、清洗、排除或阻断复核”，不负责回答“是否重要、是否锚定、是否生成、如何生成”。

`污染标签清洗规则.md` 是本规则的细化子规则，只处理候选准入时发生的污染标签清洗记录。

## 输入

- `character-evidence-ledger`
- `scene-evidence-ledger`
- `prop-evidence-ledger`
- `scene-coverage-table`

不得跳过三类 evidence ledger 直接从原始剧本重新抽取；不得重新切分场次；不得重新评级。可读取 `scene-coverage-table` 和原文证据定位来核验证据账本，但不能用生产价值提前筛掉可见事实。

## 输出

输出结果固定命名为：

- `asset-candidate-list`
- `polluted-label-cleanup-ledger`

`asset-candidate-list` 是后续 `asset-rating-result`、`global-asset-registry`、`state-variant-result` 和任务链路的候选承接结果。进入候选池不代表一定进入最终生产任务；未进入候选池的可见事实必须有污染清洗、可见排除或阻断复核说明。

`polluted-label-cleanup-ledger` 只记录候选准入时发生的污染标签处理事件。它不重复记录所有普通 `raw_label -> normalized_subject` 归一，不参与人物候选数量、S/A/B/C 数量或生产任务数量统计。

## 候选资产准入表

| 资产类型 | 可进入候选池的条件 |
|---|---|
| 人物 | 有物理出场；或虽未物理出场但后续明确有视觉生产需求；或以画像、尸体、回忆画面、梦境画面等可视觉化形式出现。 |
| 场景 | 具有明确空间身份、空间功能、可视环境差异或视觉设计需求。 |
| 道具 | 有实体出现、被使用、被特写、被交换、被打开、被损坏，或承担剧情证据、身份象征、关系信物、行动工具功能。 |

## 不进入候选池的内容

- 纯动作短语，例如“转身”“沉默”“跪下”。
- 瞬时情绪，例如“愤怒”“害怕”“冷笑”，除非后续状态拆分规则需要引用为可见状态线索。
- 人物名后拼接动作、情绪、语气、声音或瞬时表情形成的短语，例如“宋父眼眸温柔”“二夫人惊喜”“萧厌低声”“棠宁声音沙哑”“黑衣人不甘”。
- 台词前缀污染形成的人物短语，例如“皇帝点头”“棠宁喃喃”“皇后拍桌”“棠宁大惊”“棠宁轻呼”。
- 多人组合或动作组合短语，例如“两人一起”“二夫人三夫人”“二夫人三夫人一起”，除非拆回明确的单个主体。
- 镜头角度或拍摄术语。
- 没有视觉实体的抽象关系，例如“仇恨”“误会”“婚约关系”。
- 只有对白提及且没有视觉生产需求的人、地点、物。
- 无法定位个体的群体数量词，例如“众人”“百姓”“宾客”，除非需要群体模板或场面资产。
- 过于普通且没有剧情功能的环境陈设，例如普通桌椅、墙面、地砖。

## 人物候选强制准入门

进入 `asset-candidate-list` 前，人物候选必须同时满足：

1. `normalized_name` 是稳定主体名，而不是台词前缀、动作短语、情绪短语、语气短语、声音短语、瞬时表情或组合短语。
2. 能回溯到 `character-evidence-ledger` 中的具体人物证据；只出现在 `source_evidence` 原文里的修饰词不得单独成为人物。
3. 单个候选只代表一个具体主体、泛称模板或明确群体模板；不能把两个以上人物合并成一个“组合人物”。
4. 可转移称谓已完成上下文消歧；不能仅用相同称谓证明是同一个人。

不满足准入门时，不得把候选交给评级层。能识别主体的，候选只能承接干净主体；不能识别主体的，保留为证据、可见排除或阻断复核。候选准入时发生的污染拦截、拆分、无法归一或高风险污染必须按 `污染标签清洗规则.md` 进入 `polluted-label-cleanup-ledger`。

## 候选准入前污染闸门

人物 evidence row 在进入 `asset-candidate-list` 前必须先经过污染闸门：

1. 如果 `raw_label` 含动作、情绪、语气、声音、瞬时表情或组合短语，而 `normalized_subject` 是干净主体，则 `asset-candidate-list` 只能承接 `normalized_subject`。
2. 如果该 `raw_label` 有误入候选名、状态名、任务名或映射名的风险，必须进入 `polluted-label-cleanup-ledger`；如果只是普通称呼归一且无候选污染风险，不重复记录清洗台账。
3. 如果 `label_disposition` 是 `normalized_from_raw_label`，例如 `三夫人害怕`、`棠宁声音沙哑`、`萧厌轻声`，允许真实主体 `三夫人`、`宋棠宁`、`萧厌` 进入候选；污染短语本身不得进入 `raw_name`、`normalized_name`、候选 ID、资产名、状态名、任务名或分集映射。
4. 如果 `label_disposition` 是 `split_from_combination`，例如 `二夫人三夫人一起`，只能拆成 `二夫人`、`三夫人` 两个真实主体候选；组合短语本身只进入清洗台账和证据。
5. 如果无法从污染短语可靠识别主体，必须输出 `polluted-label-cleanup-ledger.handling_result: evidence_only` 或 `blocking_review`；不得把整句污染短语作为人物候选，也不得用 `C` 级兜底。

拦截和清洗不是二选一：污染短语必须被拦截在候选名之外；能归一出真实主体时，真实主体可以继续候选。不能归一时，只有污染清洗记录、证据保留、可见排除或阻断复核，不存在“污染短语候选”。

## 人物名称规范化

人物 `normalized_name` 必须是可复用的人物身份名，不得包含动作、情绪、语气、声音、台词修饰或瞬时表情。人物候选应优先从 `character-evidence-ledger.normalized_subject` 承接，污染原文只保留在 evidence ledger、`polluted-label-cleanup-ledger.raw_label` 或原文证据中。

| 原文短语 | 正确处理 |
|---|---|
| `宋父眼眸温柔` | 人物候选为 `宋父`；“眼眸温柔”只作为原文证据，不作为人物状态。 |
| `二夫人惊喜`、`二夫人惊恐`、`二夫人起身` | 人物候选为 `二夫人`；“惊喜”“惊恐”“起身”不生成新人物候选。 |
| `萧厌低声`、`棠宁声音沙哑`、`棠宁喃喃`、`棠宁大惊`、`棠宁轻呼` | 人物候选为 `萧厌`、`宋棠宁`；语气、声音和短暂反应不进入人物名。 |
| `黑衣人不甘` | 若黑衣人是泛称，则候选为 `黑衣人` 并标记 `generic`；“不甘”不生成新人物。 |
| `皇帝点头`、`皇后拍桌` | 人物候选为 `皇帝`、`皇后`；动作只保留为证据。 |
| `两人一起`、`二夫人三夫人`、`二夫人三夫人一起` | 不生成组合人物；能识别时拆为 `二夫人`、`三夫人` 等具体主体，不能识别时不进人物候选池。 |

如果无法从短语中可靠识别主体，不得把整句注册为人物；应保留在 `source_evidence` 或 `uncertainty_note`，交由审计或人工复核。

## 人物候选边界

| 情况 | 处理 |
|---|---|
| 有姓名或稳定称呼，并物理出场 | 进入候选池，`candidate_scope: named_or_stable`。 |
| 泛称人物物理出场，例如“下人”“侍卫”“丫鬟” | 可进入候选池，但必须标记 `candidate_scope: generic`，且默认只代表本场或本集泛称，不跨场合并为同一人。 |
| 多场出现相同泛称 | 不因名称频次自动升级为同一角色；只有证据显示同一个体时，后续层级才可合并。 |
| 只被提及，没有视觉生产需求 | 不进入候选池，保留在证据或可见排除说明中。 |
| 画像、尸体、梦境、回忆中可见 | 可进入候选池，并标记 `visual_form`。 |
| VO、OS、旁白或画外音中出现 | 只有声音或称呼时不进入可见人物候选池；如后续有可见画面，按可见画面证据进入。 |
| 牌位、坟墓、棺材、敬香、遗物等纪念物指代人物 | 不作为人物物理出场进入候选池；需要生产时按道具或场景候选处理。 |

## 可转移称谓消歧

- `夫人`、`四夫人`、`主子`、`大人`、`管事` 等称谓可能在不同上下文指向不同人物，不能仅按字面合并为同一人物。
- `四夫人` 如果指第一集被萧厌杀死的前任人物，应规范为 `前任四夫人` 或 `遇害四夫人`，并把适用范围限定到对应场次证据。
- 后续 `四夫人` 如果是宋棠宁的身份称谓，只能作为 `宋棠宁` 的称谓证据，不得与 `前任四夫人` 合并。
- 称谓无法消歧时，候选必须写入 `uncertainty_note`，不得注册为可生产单人脸模资产。

## 场景候选边界

| 情况 | 处理 |
|---|---|
| 明确地点或空间，例如“宋府正厅”“宫门”“牢房” | 进入候选池。 |
| 泛化地点，例如“街上”“屋内” | 可进入候选池，但标记 `candidate_scope: generic`，后续再判断是否注册或模板化。 |
| 同一地点不同空间层级 | 不在本层强行合并，保留原始空间表述和证据。 |
| 只有抽象地点关系，例如“远方”“心里” | 不进入候选池。 |

## 道具候选边界

| 情况 | 处理 |
|---|---|
| 被角色使用、交换、打开、损坏、藏匿 | 进入候选池。 |
| 承担证据、信物、身份象征、剧情机关 | 进入候选池。 |
| 普通背景陈设 | 只有在具有视觉设计需求、反复出现或影响场面理解时进入候选池。 |
| 只在对白中被提到且没有可见生产需求 | 不进入候选池。 |

## 必填字段

| 字段 | 规则 |
|---|---|
| `candidate_id` | 候选资产稳定 ID。 |
| `asset_type` | 只允许 `character`、`scene`、`prop`。 |
| `raw_name` | 准入后的干净主体原文称呼或已消歧称呼；人物 `raw_name` 不得包含动作、情绪、语气、声音、瞬时表情、组合词或未消歧可转移称谓。污染原文只能在 evidence ledger、`polluted-label-cleanup-ledger.raw_label` 或 `source_evidence` 中出现。 |
| `normalized_name` | 便于后续引用的规范名；证据不足时不要过度合并。 |
| `episode_id` | 来源集号，继承证据账本。 |
| `scene_ids` | 来源场次 ID 列表。 |
| `source_evidence_ids` | 来源 evidence ledger 证据 ID 列表。 |
| `physical_presence` | 是否有物理出场或实体出现。 |
| `candidate_scope` | 只允许 `specific`、`named_or_stable`、`generic`、`environmental`。 |
| `visual_form` | 正常出场、画像、尸体、梦境、回忆、传闻画面、陈设等；不适用时写 `normal`。 |
| `source_evidence` | 支持进入候选池的场次证据。 |
| `admission_result` | `admitted`、`evidence_only`、`visible_exclusion`、`blocking_review`。 |
| `admission_reason` | 说明为什么进入候选池，或为什么不进入候选池。 |
| `uncertainty_note` | 泛称、同名、边界不清或证据不足时必须填写。 |

候选记录必须能通过 `source_evidence_ids` 回指对应 evidence ledger 条目；若无法建立回指，必须在 `uncertainty_note` 说明并交由生产审计阻断或人工复核。不得只用自然语言 `source_evidence` 替代结构化证据 ID。

## 泛称处理

- 泛称可以进入候选池，但不得因为出现次数多就默认是同一个固定资产。
- 当不能很容易判断是同一个人、同一个空间或同一个道具时，默认不是同一个资产。
- 泛称候选项必须携带 `candidate_scope: generic` 和 `uncertainty_note`。
- 泛称人物重复出现时，优先作为群体模板、场面资产或可见排除候选处理；没有同一具体人物证据时，不得批量生成多个独立单人脸模资产。
- 后续是否注册、排除、模板化或升级，由评级和注册层基于证据决定。

## 边界

本层只判断候选资产准入：

- 不做 S/A/B/C 评级。
- 不拆分状态。
- 不合并跨集同一资产。
- 不判断是否需要人脸锚定。
- 不判断任务类型。
- 不注册资产。
- 不生成人物候选去向。
- 不生成提示词。
- 不输出最终生产表。

只使用以上列举过的字段和职责。
