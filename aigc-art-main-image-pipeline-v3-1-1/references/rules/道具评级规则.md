# 道具评级规则

## 规则目的

本文件只用于给道具 base asset 评定制作连续性等级 `S/A/B/C`。

道具评级是**物件连续性与生产复用等级**，不是物件好看程度评分。核心问题是：

> 这个道具/物件设计需要在多大范围内保持一致？

本文件只定义同一道具证据、评级顺序、等级含义和评级输出字段。

本文件不定义状态拆分、任务类型、分集映射、提示词模板或最终输出表格式。

## 输入依据

道具评级必须基于前置层级已经产出的结构化结果：

- `prop-evidence-ledger`
- `asset-candidate-list`
- `scene-coverage-table`
- 剧本原文 `source_evidence/source_locator`

`prop-evidence-ledger` 是道具评级主输入；`scene-coverage-table` 用于上下文；剧本原文定位只用于核验同一物件证据；`asset-candidate-list` 是候选承接结果，不能作为唯一事实源。禁止只根据道具名称出现频次评级。

## 评级决策顺序

道具评级必须按以下顺序判断：

0. 对照 `prop-evidence-ledger` 检查可见物件事实是否被候选承接；漏承接时阻断并指回 `asset-evidence-ledger.md` 或 `asset-candidate-admission.md`，不得静默补候选。
1. 是否能明确认定多次出现的是**同一个实体物件 / 同一个道具资产**？
2. 如果不能，默认不是同一个资产，不累计出现次数，不升级。
3. 如果能，判断是否跨多集出现。
4. 明确同一物件跨多集出现：评为 `S` 或 `A`。
5. 明确同一物件只在单集多场使用、特写、交换、揭示或发生状态变化：评为 `B`。
6. 单次、泛化、可替换、背景陈设、提及、被场景覆盖、无法确认同一物件：评为 `C`。

## 同一道具证据门

只有同一道具证据成立，才能累计出现次数。

可以作为强证据的情况：

- 稳定命名物件：特定玉佩、契约、印章、令牌、武器、书信、药瓶。
- 剧本明确追踪同一件物品。
- 有所有权转移或交接链条。
- 有独特标记、铭文、破损、血迹、封条、签名、包装、材质、状态。
- 剧情连续性依赖它是同一件东西。

证据不足的情况：

- 泛化名称：`茶杯`、`椅子`、`帘子`、`书`、`灯`、`匕首`、`信`、`药瓶`。
- 只有相同物件类型。
- 只有相同使用者阶层。
- 普通陈设多次出现。
- 不同场景中出现相似物件，但没有连续证据。

证据不足时，默认不是同一个道具资产。

## 等级定义

| 等级 | 定义 | 连续性范围 | 评级含义 |
|---|---|---|---|
| `S` | 项目级标志性或重大转折道具。同一实体物件跨多集出现，并且对重大揭示、冲突或人物/世界身份有核心作用。 | 全项目 | 后续作为全局道具资产复用。 |
| `A` | 项目级复用道具。同一实体物件跨多集出现，但频次或剧情依赖低于 S。 | 全项目或主要段落 | 后续作为全局道具资产复用。 |
| `B` | 单集级重要道具。同一物件在单集多场使用、出现特写、交换、阅读、破损、打开或发生可见状态变化。 | 单集或短段落 | 后续作为本集/短段落道具资产。 |
| `C` | 泛化陈设、可替换物、提及物、被场景覆盖物，或无法确认同一物件的道具。 | 不固定 | 通常不需要独立物件连续。 |

## S 与 A 的区别

使用 `S` 的情况：

- 全项目高频出现。
- 重大揭示、冲突、法理、威胁、亲密关系或身份识别依赖它。
- 与人物或世界观强绑定。
- 有多个可见状态或大量特写复用。

使用 `A` 的情况：

- 同一物件跨多集出现。
- 频次、状态复杂度或剧情依赖低于 `S`。

不得因为普通物件出现次数多就升级，例如杯子、椅子、帘子、碗、书、灯、泛化匕首。

## B 级规则

满足以下情况时评为 `B`：

| 情况 | 评级 |
|---|---|
| 一封信在一集中被阅读、隐藏、揭示 | `B` |
| 一把武器在一集多场出现，需要明确设计 | `B` |
| 令牌、药瓶、印章、契约、发簪在一集中有特写 | `B` |
| 某件独特物品属于单集级人物，并影响本集视觉连续 | `B` |

如果同一道具后续跨集出现，并且同一物件证据成立，应退回评级层，升级为 `A`。

## C 级规则

没有独立生产连续需求，或同一物件证据不足时，评为 `C`。

| 情况 | 评级/处理 |
|---|---|
| 泛化杯子、碗、椅子、帘子、植物 | `C`，说明由场景陈设覆盖 |
| 只在对白中提及、没有实体出现 | `C`，写明无视觉生产需求 |
| 没有特写或动作功能的可替换背景物 | `C` |
| 多集出现“匕首”，但无证据证明是同一把武器 | `C` 或拆成本地道具行，不升级 |

## 评级输出字段

`asset-rating-result` 中的道具评级项只使用以下字段：

| 字段 | 是否必填 | 允许值/规则 | 含义 |
|---|---:|---|---|
| `base_asset_id` | 是 | 稳定道具资产 ID | 被评级的道具资产 |
| `asset_type` | 是 | `prop` | 资产类型 |
| `asset_name` | 是 | 规范化道具名 | 后续注册和映射引用名 |
| `priority_level` | 是 | `S/A/B/C` | 物件连续性/复用等级 |
| `same_asset_evidence` | 是 | `strong/medium/weak/conflict` | 判断是否同一物件的证据强度 |
| `episode_span` | 是 | 集数列表或单集 | 道具确认出现范围 |
| `scene_ids` | 是 | 场次 ID 列表 | 评级依据场次 |
| `rating_evidence_ids` | 是 | `prop-evidence-ledger.evidence_id` 列表 | 支持本次评级的结构化证据回指 |
| `rating_reason` | 是 | 文本 | 为什么评为该等级 |
| `source_evidence` | 是 | 剧本证据 | 支持评级的证据 |
| `rating_uncertainty_note` | 条件必填 | 文本或空 | 同一物件证据不足、冲突或降级原因 |

`rating_evidence_ids` 不得为空；不得只写自然语言 `source_evidence` 而缺少 evidence ledger 回指。若评级所需证据尚未进入 `prop-evidence-ledger`，必须阻断回 `asset-evidence-ledger.md` 或 `asset-candidate-admission.md`。

## 边界

- 不拆分道具状态。
- 不判断 `task_type`。
- 不设置 `review_status`。
- 不生成提示词。
- 不生成分集映射。
- 不重新扫 PDF。
- 不重新切分集场。
- 不静默补写候选；候选或证据账本漏项必须阻断回上游。

只使用以上列举过的字段和职责。
