# 生产审计规则

## 规则目的

本文件只用于 `production-audit.md` 的生产前一致性审计。

生产审计的核心问题是：

> 前序结构化结果是否完整、一致、可被提示词生成层安全读取？

本文件只定义审计对象、审计问题类型、严重程度和 `production-audit-result` 输出字段。

本文件不重新抽取资产、不重新拆状态、不重新评级、不重写提示词、不直接修复注册表或分集映射。

## 审计对象

| 审计对象 | 检查内容 |
|---|---|
| `scene-coverage-table` | 场次事实是否被候选资产、映射或可见排除记录覆盖 |
| `asset-candidate-list` | 候选资产是否都有注册、合并、排除或审核去向 |
| `state-variant-result` | 状态是否挂到正确 base asset，是否存在状态孤儿 |
| `asset-rating-result` | 每个 base asset 是否有唯一 `priority_level`，状态是否继承等级 |
| `global-asset-registry` | 资产 ID、状态 ID、别名合并是否稳定 |
| `global-task-draft` | 应生产资产是否都有任务行，`task_id`、`task_type`、`review_status`、`anchor_task_id` 是否完整合法 |
| `character-disposition-ledger` | 每个人物候选是否有最终去向，B/C 是否未被静默丢失，去向是否引用真实任务或可见排除记录 |
| `polluted-label-cleanup-ledger` | 污染标签是否只在清洗记录或原文证据中出现，字段是否合法，目标候选是否可追踪 |
| `episode-usage-map-draft` | 分集映射是否能引用到存在的 `task_id` |
| `character-anchor-result` | S/A 人物是否有锚点任务，状态图是否依赖锚点；不得当作完整人物清单 |

## 问题类型

| 问题类型 | 审计动作 |
|---|---|
| 缺必要字段 | 输出审计问题，指向负责字段的上游层级 |
| `task_id` 断链 | 输出阻断问题，要求回到全局任务草稿或分集映射层处理 |
| S/A 人物缺锚点 | 输出阻断问题，要求回到角色锚定层处理 |
| 状态孤儿 | 输出阻断问题，要求回到状态拆分或资产注册层处理 |
| 候选资产无去向 | 输出覆盖问题，要求回到资产注册层处理 |
| 应生产资产缺任务 | 输出阻断问题；要求回到资产注册层按 `全局资产任务生成规则.md` 补齐 `global-task-draft` |
| 人物候选无最终去向 | 输出阻断问题，要求回到资产注册层补齐 `character-disposition-ledger` |
| B/C 可生产人物缺 `text_to_image` 任务 | 输出阻断问题；B/C 可生产人物必须有生产任务，不能只存在于人物候选去向台账 |
| B/C 人物无闭环 | 输出阻断问题；B/C 人物必须有真实生产任务、状态任务、模板任务、别名、有效分组、排除或非视觉去向 |
| B 级人物数量异常为 0 | 输出警告或阻断问题；必须要求回到评级结果核查是否把单集多片段风险人物误判为 C，不能用空 B 结果直接通过 |
| 锚点结果误作完整人物清单 | 输出阻断问题，要求回到输出契约或资产注册层，不能用 `character-anchor-result` 代替人物候选台账 |
| 人物去向引用断链 | 输出阻断问题；`task_id`、`alias_to`、`grouped_into` 必须能引用到存在目标 |
| 人工审核状态替代任务 | 输出阻断问题；`manual_review_required` 只能作为任务行 `review_status`，不能替代应生产资产的 `task_id` |
| 空分组替代任务 | 输出阻断问题；`grouped_into` 必须指向真实模板任务、集合任务或可见排除记录 |
| 泛称人物粗暴合并 | 输出警告或阻断问题；缺少同一人物证据时要求回到资产注册层拆分、分组、模板化或排除 |
| 分集映射缺引用 | 输出映射问题，要求回到分集映射生成位置处理 |
| 审核状态与任务字段冲突 | 输出字段问题，要求回到任务类型字段审核规则处理 |
| 疑似场次标题未解析 | 输出阻断问题；要求回到 `script-intake.md` 修正场次边界，例如 `65-2（闪回）...` 不得残留在上一场文本中 |
| 污染人物名 | 输出阻断问题；人物 `normalized_name`、`asset_name`、`state_name`、`task_id` 对应名称或分集映射名称不得包含动作、情绪、语气、声音、瞬时表情、组合词或未消歧可转移称谓 |
| 污染清洗记录缺失或断链 | 输出阻断问题；发现污染标签但没有 `polluted-label-cleanup-ledger` 记录，或清洗记录字段、枚举、`target_candidate_id` 不合法 |
| 污染清洗记录误计数 | 输出阻断问题；清洗记录不得计入人物候选数、S/A/B/C 数或生产任务数 |
| 人物状态主体不明 | 输出阻断问题；状态不能只由场级关键词触发并分配给多个人物 |
| 非法人物状态 | 输出阻断问题；纯动作、纯情绪、语气、声音、短暂表情不得成为状态任务 |
| 泛称人物任务爆炸 | 输出阻断问题；泛称、群体或背景人物不得仅因跨场/跨集重复出现而生成大量独立单人脸模任务 |
| A/S 评级缺少可见脸模证据 | 输出阻断问题；A/S 不能只由 VO、提及、牌位、坟墓、棺材或纪念物支撑 |

生产审计只能给出 `required_action`，不得在本层直接补字段、补注册、补人物任务、改评级、改状态或重写提示词。

## 严重程度

| `severity` | 含义 | 是否可进入提示词生成 |
|---|---|---:|
| `blocking` | 会导致任务断链、锚点缺失、字段无法读取或资产去向缺失。 | 否 |
| `warning` | 可以生成，但需要人工确认或后续复核。 | 是 |
| `info` | 不影响生产，只用于说明。 | 是 |

## 输出要求

`production-audit-result` 至少包含：

- `audit_status`
- `issue_id`
- `issue_type`
- `affected_layer`
- `affected_id`
- `severity`
- `required_action`
- `suggested_review_status_update`
- `review_reason`

审计结果必须被 `prompt-generation.md`、`output-contracts.md` 和 `final-quality.md` 读取。

## 任务完整性审计

生产审计必须用 `asset-rating-result`、`global-asset-registry`、`global-task-draft`、`character-disposition-ledger` 和 `polluted-label-cleanup-ledger` 交叉检查：

- S/A/B/C 已注册且应生产的角色、场景、道具资产是否都有 `global-task-draft` 行。
- B 级人物基础形象是否都有 `text_to_image` 任务和 `episode_id`。
- B 级人物数量为 0 时，是否有评级层或审计摘要说明：候选人物中没有明确同一具体人物、只在单集出现、且不能证明全部可见戏份处于同一个最长 15 秒单片段的人物；如无说明，必须标记为至少 `warning`，对长剧或人物候选较多项目可标记为 `blocking`。
- C 级可视单人或模板人物如果保留为资产，是否有 `text_to_image` 任务或合法可见排除记录。
- C 级人物是否符合硬分流：不能确认同一具体人物；或明确同一具体人物只在单集出现，且全部可见戏份能被证明处于同一个最长 15 秒单片段内。背景、群体、泛称、只提及只能作为“不能确认同一具体人物”或“无视觉生产需求”的证据，不得绕过硬分流把单集多片段同一具体人物降为 C。
- `production_task`、`state_task`、`template_task` 去向是否都能在 `global-task-draft` 找到对应 `task_id`。
- `manual_review_required` 是否只出现在任务行 `review_status` 或不可形成任务的去向记录中。
- `grouped_into` 是否指向真实模板任务、集合任务、别名目标或可见排除记录。
- `script-intake-result` 或 `scene-coverage-table` 中是否存在疑似未解析场次标题；存在时不得进入提示词生成。
- `scene-coverage-table.character_mentions`、`asset-candidate-list`、`global-asset-registry`、`character-disposition-ledger`、`global-task-draft`、`episode-usage-map-draft` 或状态任务中是否存在污染名。污染名包括“人物名 + 动作/情绪/语气/声音/瞬时表情”、组合短语、数量短语和未消歧可转移称谓，例如“宋父眼眸温柔”“二夫人惊喜”“二夫人惊恐”“三夫人害怕”“萧厌低声”“棠宁声音沙哑”“棠宁喃喃”“棠宁大惊”“棠宁轻呼”“皇后拍桌”“皇帝点头”“两人一起”“二夫人三夫人”“二夫人三夫人一起”。
- 污染短语允许只保留在 `source_evidence` 原文证据和 `polluted-label-cleanup-ledger.raw_label` 中；一旦进入结构化人物名、状态名、任务名、人物候选去向台账名或映射名，必须阻断。
- `polluted-label-cleanup-ledger` 是否包含必填字段，`cleanup_type` 和 `handling_result` 是否使用合法枚举，`target_candidate_id` 是否能在 `asset-candidate-list` 中找到。
- 如果污染标签被识别但没有清洗记录，必须阻断并指回 `asset-registry.md`；如果污染标签位于 `asset-candidate-list`，必须阻断并指回 `asset-extraction.md`。
- `polluted-label-cleanup-ledger` 是否被错误计入人物候选数量、S/A/B/C 数量、生产任务数量或 `review_status` 统计。
- `四夫人`、`夫人`、`主子`、`管事` 等可转移称谓是否已经绑定具体人物和时间范围。若 `四夫人` 同时可能指 E01 前任/遇害人物和后续宋棠宁称谓，但未消歧，必须阻断。
- 人物状态任务是否能回溯到明确主体证据；如果同一场的状态关键词被分配给多个无主体证据人物，必须阻断。
- A/S 人物的跨集依据是否包含可见脸模生产出场；如果只由 `voice_only`、`mentioned_only`、`memorial_reference` 或等价证据支撑，必须阻断并指回评级层。
- C 泛称或群体人物是否被大量拆成独立单人 `production_task`；如果没有模板差异、场面差异或同一具体人物证据，必须阻断并指回资产注册层模板化、分组或排除。

以上检查失败时，`severity` 必须为 `blocking`。污染源位于 `character_mentions` 时，`affected_layer` 指向 `scene-coverage.md`；位于 `asset-candidate-list` 时，指向 `asset-extraction.md`；位于 `state-variant-result` 或状态任务时，指向 `state-variant.md`；位于注册表、任务草稿、人物去向台账、污染清洗记录或分集映射时，指向 `asset-registry.md`。

疑似场次标题未解析时，`affected_layer` 必须指向 `script-intake.md`。A/S 评级缺少可见脸模证据时，`affected_layer` 必须指向 `asset-rating.md`。人物状态主体不明时，`affected_layer` 必须指向 `state-variant.md`。

## 边界

- 不重新抽取资产。
- 不重新拆状态。
- 不重新评级。
- 不建立锚点。
- 不生成提示词。
- 不修改最终 Markdown 输出。

只使用以上列举过的字段和职责。
