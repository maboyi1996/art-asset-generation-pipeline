# 生产审计规则

## 规则目的

本文件只用于 `production-audit.md` 的生产前一致性审计。

生产审计的核心问题是：

> 前序结构化结果是否完整、一致、可被提示词生成层安全读取？

本文件只定义审计对象、审计问题类型、严重程度和 `production-audit-result` 输出字段。

本文件不重新抽取资产、不重新拆状态、不重新评级、不重写提示词、不直接修复注册表或分集映射。

## 审计对象

| 审计对象 | 检查内容 |
|---|---|
| `scene-coverage-table` | 场次事实是否被候选资产、映射或可见排除记录覆盖 |
| `asset-candidate-list` | 候选资产是否都有注册、合并、排除或审核去向 |
| `state-variant-result` | 状态是否挂到正确 base asset，是否存在状态孤儿 |
| `asset-rating-result` | 每个 base asset 是否有唯一 `priority_level`，状态是否继承等级 |
| `global-asset-registry` | 资产 ID、状态 ID、别名合并是否稳定 |
| `global-task-draft` | `task_id`、`task_type`、`review_status`、`anchor_task_id` 是否完整合法 |
| `episode-usage-map-draft` | 分集映射是否能引用到存在的 `task_id` |
| `character-anchor-result` | S/A 人物是否有锚点任务，状态图是否依赖锚点 |

## 问题类型

| 问题类型 | 审计动作 |
|---|---|
| 缺必要字段 | 输出审计问题，指向负责字段的上游层级 |
| `task_id` 断链 | 输出阻断问题，要求回到全局任务草稿或分集映射层处理 |
| S/A 人物缺锚点 | 输出阻断问题，要求回到角色锚定层处理 |
| 状态孤儿 | 输出阻断问题，要求回到状态拆分或资产注册层处理 |
| 候选资产无去向 | 输出覆盖问题，要求回到资产注册层处理 |
| 分集映射缺引用 | 输出映射问题，要求回到分集映射生成位置处理 |
| 审核状态与任务字段冲突 | 输出字段问题，要求回到任务类型字段审核规则处理 |

生产审计只能给出 `required_action`，不得在本层直接补字段、补注册、改评级、改状态或重写提示词。

## 严重程度

| `severity` | 含义 | 是否可进入提示词生成 |
|---|---|---:|
| `blocking` | 会导致任务断链、锚点缺失、字段无法读取或资产去向缺失。 | 否 |
| `warning` | 可以生成，但需要人工确认或后续复核。 | 是 |
| `info` | 不影响生产，只用于说明。 | 是 |

## 输出要求

`production-audit-result` 至少包含：

- `audit_status`
- `issue_id`
- `issue_type`
- `affected_layer`
- `affected_id`
- `severity`
- `required_action`
- `suggested_review_status_update`
- `review_reason`

审计结果必须被 `prompt-generation.md`、`output-contracts.md` 和 `final-quality.md` 读取。

## 边界

- 不重新抽取资产。
- 不重新拆状态。
- 不重新评级。
- 不建立锚点。
- 不生成提示词。
- 不修改最终 Markdown 输出。

只使用以上列举过的字段和职责。
