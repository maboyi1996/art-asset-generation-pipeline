# 资产抽取边界规则

## 规则目的

本文件约束 `asset-candidate-list` 的准入边界。

本层的目的不是给资产定级，而是从 `scene-coverage-table` 中筛出“可能需要进入后续制作判断”的人物、场景和道具候选项，同时过滤明显不是资产的内容。它负责回答“是否值得进入候选池”，不负责回答“是否重要、是否锚定、是否生成、如何生成”。

## 输入

- `scene-coverage-table`

不得跳过场次覆盖表直接从原始剧本重新抽取；不得重新切分场次；不得重新评级。

## 输出

输出结果固定命名为 `asset-candidate-list`。

`asset-candidate-list` 是后续 `state-variant-result`、`asset-rating-result` 和 `global-asset-registry` 的候选来源。进入候选池不代表一定进入最终生产任务。

## 候选资产准入表

| 资产类型 | 可进入候选池的条件 |
|---|---|
| 人物 | 有物理出场；或虽未物理出场但后续明确有视觉生产需求；或以画像、尸体、回忆画面、梦境画面等可视觉化形式出现。 |
| 场景 | 具有明确空间身份、空间功能、可视环境差异或视觉设计需求。 |
| 道具 | 有实体出现、被使用、被特写、被交换、被打开、被损坏，或承担剧情证据、身份象征、关系信物、行动工具功能。 |

## 不进入候选池的内容

- 纯动作短语，例如“转身”“沉默”“跪下”。
- 瞬时情绪，例如“愤怒”“害怕”“冷笑”，除非后续状态拆分规则需要引用为可见状态线索。
- 镜头角度或拍摄术语。
- 没有视觉实体的抽象关系，例如“仇恨”“误会”“婚约关系”。
- 只有对白提及且没有视觉生产需求的人、地点、物。
- 无法定位个体的群体数量词，例如“众人”“百姓”“宾客”，除非需要群体模板或场面资产。
- 过于普通且没有剧情功能的环境陈设，例如普通桌椅、墙面、地砖。

## 人物抽取边界

| 情况 | 处理 |
|---|---|
| 有姓名或稳定称呼，并物理出场 | 进入候选池，`candidate_scope: named_or_stable`。 |
| 泛称人物物理出场，例如“下人”“侍卫”“丫鬟” | 可进入候选池，但必须标记 `candidate_scope: generic`，且默认只代表本场或本集泛称，不跨场合并为同一人。 |
| 多场出现相同泛称 | 不因名称频次自动升级为同一角色；只有证据显示同一个体时，后续层级才可合并。 |
| 只被提及，没有视觉生产需求 | 不进入候选池，保留在场次覆盖证据中即可。 |
| 画像、尸体、梦境、回忆中可见 | 可进入候选池，并标记 `visual_form`。 |

## 场景抽取边界

| 情况 | 处理 |
|---|---|
| 明确地点或空间，例如“宋府正厅”“宫门”“牢房” | 进入候选池。 |
| 泛化地点，例如“街上”“屋内” | 可进入候选池，但标记 `candidate_scope: generic`，后续再判断是否注册或模板化。 |
| 同一地点不同空间层级 | 不在本层强行合并，保留原始空间表述和证据。 |
| 只有抽象地点关系，例如“远方”“心里” | 不进入候选池。 |

## 道具抽取边界

| 情况 | 处理 |
|---|---|
| 被角色使用、交换、打开、损坏、藏匿 | 进入候选池。 |
| 承担证据、信物、身份象征、剧情机关 | 进入候选池。 |
| 普通背景陈设 | 只有在具有视觉设计需求、反复出现或影响场面理解时进入候选池。 |
| 只在对白中被提到且没有可见生产需求 | 不进入候选池。 |

## 必填字段

| 字段 | 规则 |
|---|---|
| `candidate_id` | 候选资产稳定 ID。 |
| `asset_type` | 只允许 `character`、`scene`、`prop`。 |
| `raw_name` | 原文名称或称呼。 |
| `normalized_name` | 便于后续引用的规范名；证据不足时不要过度合并。 |
| `episode_id` | 来源集号，继承 `scene-coverage-table`。 |
| `scene_ids` | 来源场次 ID 列表。 |
| `physical_presence` | 是否有物理出场或实体出现。 |
| `candidate_scope` | 只允许 `specific`、`named_or_stable`、`generic`、`environmental`。 |
| `visual_form` | 正常出场、画像、尸体、梦境、回忆、传闻画面、陈设等；不适用时写 `normal`。 |
| `source_evidence` | 支持进入候选池的场次证据。 |
| `extraction_reason` | 说明为什么进入候选池。 |
| `uncertainty_note` | 泛称、同名、边界不清或证据不足时必须填写。 |

## 泛称处理

- 泛称可以进入候选池，但不得因为出现次数多就默认是同一个固定资产。
- 当不能很容易判断是同一个人、同一个空间或同一个道具时，默认不是同一个资产。
- 泛称候选项必须携带 `candidate_scope: generic` 和 `uncertainty_note`。
- 后续是否注册、排除、模板化或升级，由评级和注册层基于证据决定。

## 边界

本层只判断候选资产准入：

- 不做 S/A/B/C 评级。
- 不拆分状态。
- 不合并跨集同一资产。
- 不判断是否需要人脸锚定。
- 不判断任务类型。
- 不生成提示词。
- 不输出最终生产表。

只使用以上列举过的字段和职责。

## 示例

| 场次覆盖事实 | 抽取结果 |
|---|---|
| “棠宁推门而入” | 人物“棠宁”进入候选池，`candidate_scope: named_or_stable`。 |
| “几个下人抬着箱子进来” | “下人”可进入候选池但标记 `generic`，不跨场默认同一人；“箱子”如被使用或承载剧情功能则进入道具候选。 |
| “众人议论纷纷” | “众人”通常不进入候选池，除非需要群体模板。 |
| “宋府正厅灯火通明” | 场景“宋府正厅”进入候选池；“灯火通明”保留为可见环境线索，不单独作为道具资产。 |
| “她想起那枚玉佩”但画面没有出现玉佩 | 若只是对白/心理提及，不进入候选池；若后续明确要视觉呈现，进入道具候选并说明依据。 |
