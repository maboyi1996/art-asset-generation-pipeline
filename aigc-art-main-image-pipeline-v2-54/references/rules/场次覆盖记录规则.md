# 场次覆盖记录规则

## 规则目的

本文件约束 `scene-coverage-table` 的记录方式。

本层的目的不是决定资产是否成立，而是基于 `script-intake-result` 对每一个场次做完整事实覆盖，记录场内可见的人物、地点、道具、状态线索和证据。它是后续资产抽取、状态拆分、评级、注册和审计的事实台账。

## 输入

- `script-intake-result`
- 原始剧本文本块和对应 `source_locator`

不得绕过 `script-intake-result` 重新切分集、场或文本范围。

## 输出

输出结果固定命名为 `scene-coverage-table`。

`scene-coverage-table` 必须覆盖 `script-intake-result.scene_index` 中的每一个场次。即使某场没有新增资产线索，也必须保留记录。

## 每场必填字段

| 字段 | 规则 |
|---|---|
| `episode_id` | 继承 `script-intake-result`。 |
| `scene_id` | 继承 `script-intake-result`。 |
| `scene_heading` | 原场次标题或内部标题。 |
| `location_text` | 场内原始地点表述，保持剧本文字，不做场景资产合并。 |
| `time_text` | 场内可见时间信息，例如白天、夜晚、翌日、雨夜；无明确信息写 `unknown`。 |
| `character_mentions` | 场内出现或被提及人物，必须区分可见真人、闪回/梦境可见、画像/尸体可见、对白提及、VO、纪念物指代。 |
| `prop_mentions` | 场内出现、被使用、被特写、被交换、被打开、被损坏或承担证据功能的道具。 |
| `visible_state_clues` | 服装、伤病、脏污、血迹、妆发、年龄阶段、昼夜、天气、破损、打开/关闭等可见变化线索；人物状态线索必须能指向明确主体。 |
| `scene_summary` | 只写场内事实，不写制作判断，不写剧情评价。 |
| `source_evidence` | 支持本场记录的原文证据，可以是短引文、页码、段落、行号或文本块编号。 |
| `coverage_notes` | 场内信息缺失、文本不清、疑似跨场延续、无新增资产线索等备注。 |

## 人物记录要求

| 情况 | 记录方式 |
|---|---|
| 正常时空内明确可见真人出场 | 记录为 `presence_type: physical_visible`。 |
| 闪回、回忆、梦境、想象画面中可见真人 | 记录为 `presence_type: memory_visible`、`dream_visible` 或等价可见叙事类型。 |
| 画像、尸体、遗体、通缉画像等可见但不是正常真人出场 | 记录为 `presence_type: portrait_visible`、`corpse_visible` 或等价可见媒介类型。 |
| VO、OS、画外音、旁白中出现人物声音或称呼 | 记录为 `presence_type: voice_only`。 |
| 只在对白中被提及 | 记录为 `presence_type: mentioned_only`。 |
| 牌位、坟墓、棺材、灵堂祭拜、遗物等指向人物 | 记录为人物相关剧情事实或道具/场景线索，`presence_type: memorial_reference`，不得当作真人可见出场。 |
| 传闻中出现但没有可见画面 | 记录为 `presence_type: rumor_mentioned`。 |
| “下人”“侍卫”“丫鬟”等泛称 | 按本场事实记录为泛称，不默认与其他场次是同一个人。 |
| 姓名和称呼同时出现 | 保留原文称呼，不在本层合并别名。 |

只有 `physical_visible`、`memory_visible`、`dream_visible` 这类包含可见人脸生产形态的记录，才可作为后续人物资产和脸模连续性判断的可见出场事实。`voice_only`、`mentioned_only`、`memorial_reference`、`rumor_mentioned` 不得在后续评级中被累计为跨集真人脸模出场。

## 状态线索记录要求

- 人物状态线索必须记录主体，例如“宋父--受伤”“二夫人--衣衫凌乱”；无法指向主体时，只能作为场内备注或送审线索，不得自动分配给本场所有人物。
- 场景状态线索只能描述空间条件，例如夜晚、火灾、封闭、仪式布置。
- 道具状态线索只能描述物体条件，例如染血、破损、打开、封存。
- 纯动作、纯情绪、语气、声音、短暂表情，例如“起身”“低声”“惊喜”“担忧”“不甘”“声音沙哑”，不得作为人物状态线索记录，除非文本明确形成稳定可见外观变化。

## 场景记录要求

- `location_text` 只记录原文地点和空间描述。
- 本层不判断“宋府”“宋家”“宋府正厅”是否为同一场景资产。
- 如果一场中有多个空间变化，应在 `coverage_notes` 中说明，必要时列出多个地点表述。

## 道具记录要求

- 记录实体道具和可见陈设，但不要在本层判断其生产优先级。
- 普通陈设如果只是环境背景，可以记录为场内事实；是否进入候选资产由 `资产抽取边界规则.md` 决定。
- 被反复使用、被特写、承载证据或推动剧情的道具，应保留清楚证据。

## 边界

本层只记录场内事实：

- 不判断候选资产是否成立。
- 不合并同名人物、地点、道具。
- 不累计跨场或跨集出现次数。
- 不判断是否同一资产。
- 不做 S/A/B/C 评级。
- 不拆分生产状态。
- 不判断任务类型。
- 不生成提示词。

只使用以上列举过的字段和职责。

## 覆盖原则

- 每个已解析场次都必须有一条覆盖记录。
- 没有新增人物、场景、道具的场次，也要写明“无新增资产线索”或“仅延续既有资产线索”。
- 不因为某场信息少而省略记录。
- 不因为某人物或道具看似不重要而省略其场内事实。

## 示例

| 原文情况 | 正确记录 |
|---|---|
| “几个下人匆匆进来” | 记录泛称人物“下人”，`presence_type: physical_visible`，不判断为同一个固定角色。 |
| “宋夫人提起萧厌” | 宋夫人为物理出场；萧厌为 `mentioned_only`。 |
| “（VO）宋父温柔：宁宁乖……” | 记录宋父为 `voice_only`，不作为真人可见出场。 |
| “为宋父宋母敬香，祠堂里有棺材” | 宋父宋母记录为 `memorial_reference`，棺材/牌位按道具或场景线索处理。 |
| “桌上放着染血帕子” | 记录道具“染血帕子”和状态线索“染血”。 |
| 某场只有对话没有新物件 | 仍保留该场记录，并在 `coverage_notes` 写明无新增资产线索。 |
