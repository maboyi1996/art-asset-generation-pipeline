# 人物评级规则

## 规则目的

本文件只用于给人物 base asset 评定制作连续性等级 `S/A/B/C`。

人物评级不是大众意义上的剧情重要性评分，而是**脸模连续性等级**。核心问题是：

> 这个人物的脸需要在多大范围内保持一致？

本文件只定义同一人物证据、评级顺序、等级含义和评级输出字段。

本文件不定义状态拆分、任务类型、锚点流程、分集映射、提示词模板或最终输出表格式。

## 输入依据

人物评级必须基于前置层级已经产出的结构化结果：

- `asset-candidate-list`
- `state-variant-result`
- `scene-coverage-table`
- 剧本原文证据

禁止只根据名称出现频次评级。

## 评级决策顺序

人物评级必须按以下顺序判断：

1. 是否能明确认定候选是**同一个具体人物**？
2. 如果不能，默认不是同一个人物，不累计出现次数，不升级，评为 `C` 或拆成不同 C/模板行。
3. 如果能，判断是否存在跨多集可见脸模生产出场。
4. 明确同一人跨多集可见脸模生产出场：评为 `S` 或 `A`。
5. 明确同一人只在单集内出现：先判断是否有 `C` 级正向证明。
6. 能证明只是单片段、单镜、一次性功能、背景、群体、泛称、只提及或无需单人脸模连续：评为 `C`。
7. 明确同一人只在单集内出现，且不能证明是单片段人物：评为 `B`。

## 同一人物证据门

只有同一人物证据成立，才能累计出现次数。

可以作为强证据的情况：

- 稳定姓名、封号、职务加唯一归属，例如“宋夫人”“陆皇后”。
- 剧本明确写明“同一人”“仍是他/她”“上次那人”。
- 与 S/A 人物存在稳定关系绑定，例如同一个母亲、丈夫、随从、贴身侍女。
- 有连续事件链、伤病状态、身份关系或专属道具证明是同一人。
- 角色在上下文中被明确追踪。

证据不足的情况：

- 只有相同泛称：`下人`、`丫鬟`、`侍卫`、`小厮`、`黑衣人`、`乞丐`、`太医`、`百姓`。
- 只有相同职业或阶层。
- 只有相同性别、年龄段。
- 只有相似通用服装。
- 不同集或不同场只出现同类角色，没有明确连续证据。

证据不足时，默认不是同一个人。

## 可见出场计数口径

人物连续性评级只统计同一具体人物的可见脸模生产出场。

可以计入人物脸模连续性的情况：

- 正常时空内的真人可见出场。
- 闪回、回忆、梦境、想象画面中需要生产可见人脸的出场。
- 画像、尸体、遗体等需要生产同一人物可见形象时，可计入可见形态证据，但必须在 `source_evidence` 写明 `visual_form`，并由评级层判断是否真的需要同一脸模连续。

不得计入跨集真人脸模出场的情况：

- VO、OS、旁白、画外音。
- 只在对白、心理活动或旁白中被提及。
- 牌位、坟墓、棺材、敬香、祠堂祭拜、遗物等纪念物指向。
- “父亲”“母亲”“阿爹”“阿娘”等亲属称呼在不同上下文中的泛化提及。
- 只有剧情关系存在，没有可见人物形象生产需求。

这些证据可以作为剧情、道具、场景或关系证据，但不能让人物从 `B/C` 升为 `A/S`。如果需要生产牌位、坟墓、棺材、遗物，应转入场景或道具规则，而不是作为人物跨集出场。

## 等级定义

| 等级 | 定义 | 连续性范围 | 评级含义 |
|---|---|---|---|
| `S` | 项目级主要角色。同一个人跨多集可见脸模生产出场，且高频、主线长期依赖、通常有多状态资产。 | 全项目固定脸 | 后续必须建立项目级人脸锚点。 |
| `A` | 项目级连续关键角色。同一个人跨多集可见脸模生产出场，但频次或主线权重低于 S。剧情重要性可以低，跨集脸模连续性才是决定因素。 | 全项目固定脸 | 后续必须建立项目级人脸锚点。 |
| `B` | 单集级连续角色。明确同一人只在单集内出现，但不是单片段人物，需要本集内固定脸。 | 单集内固定脸 | 后续保持本集内脸模一致。 |
| `C` | 单片段、单镜、背景、功能、模板、群体、泛称、只提及人物，或无法确认同一具体人物。 | 不固定脸 | 通常不需要脸模连续。 |

## S 与 A 的区别

`S` 和 `A` 都是需要人脸锚定的项目级角色。

使用 `S` 的情况：

- 全项目高频出现。
- 主线或核心关系长期依赖。
- 有大量后续状态图依赖。
- 观众识别该人物是项目级核心需求。

使用 `A` 的情况：

- 同一个人跨多集出现。
- 跨集证据必须是可见脸模生产出场，不得只由 VO、提及、纪念物或亲属称呼支撑。
- 频次、主线权重或状态数量低于 `S`。
- 即使剧情重要性较低，只要明确同一人跨多集出现，也必须进入 `A`，因为制作上需要固定脸模。

示例：

| 情况 | 评级 |
|---|---|
| 同一个小乞丐在 E01 露一次脸，在最后一集又露一次脸，且证据明确证明是同一个人 | `A` |
| E01 有“小乞丐”，最后一集也有“小乞丐”，但没有证据证明是同一个人 | `C` 或拆成不同 C/模板行，不升级 |

## B 级规则

满足以下条件时评为 `B`：

- 能明确判断是同一个人。
- 所有确认出现都在同一集内。
- 不能证明该人物只是单片段或单镜人物。
- 如果脸变了，会影响本集视觉连续性。
- 不需要项目级人脸锚定。

人物评级层不做 1:1 文字分镜，也不读取文字分镜结果。评级层只基于剧本和场次覆盖做多片段风险预判。生产片段的现实参考是 4-15 秒；如果一个明确同一人的单集角色无法证明能在一个 4-15 秒片段内完成全部可见戏份，应评为 `B`。

以下信号应判为 `B`，不要求已经完成文字分镜：

- 同一具体人物在同一集多个场次出现。
- 同一具体人物在同一场内有多轮动作、对话、互动或反应。
- 同一具体人物在同一场内存在出场、行动、被质问、反应、离场等连续过程。
- 同一具体人物有可见状态变化，例如受伤、跪下、被押、惊慌、换装、脏污等。
- 同一具体人物参与关键剧情推进，明显不是一个 4-15 秒片段内的一次性功能露出。

示例：

| 情况 | 评级 |
|---|---|
| 某有名证人在 E05 多场出现，且只服务 E05 | `B` |
| 固定大夫在 E12 多场给关键人物诊治 | `B` |
| 某传旨太监在同一集多次出现，且明确是同一个人 | `B` |
| 某有名或有稳定描述的丫鬟只在一集内多场出现 | `B` |
| 某少年角色只在一个回忆场出现，但有被带走、目击、反应等连续过程，无法证明只占一个 4-15 秒片段 | `B` |
| 某太监只在一场出现，但有传旨、被质问、回应、退下等连续动作 | `B` |

如果一个 `B` 级人物后续又在其他集出现，并且同一人物证据成立，应退回评级层，升级为 `A`。

## C 级规则

`C` 级需要有正向证明：该候选是单片段、单镜、一次性功能、背景、群体、泛称、只提及、无法确认同一具体人物，或生产上无需单人脸模连续。

如果候选已经能明确认定为同一个具体人物，并且只在单集内出现，但不能证明其只是单片段人物，不得直接评为 `C`，应评为 `B`。

示例：

| 情况 | 评级/处理 |
|---|---|
| `下人若干`、`丫鬟们`、`侍卫们`、`百姓` | `C`，群体/模板或排除候选 |
| 某仆人只在一个场次执行一次动作 | `C` |
| `黑衣人若干`，无稳定首领或命名个体 | `C`，不做人脸锚点 |
| 只被提及、没有视觉生产需求的人物 | `C`，写明无视觉生产需求 |
| 某小厮只开门或递物一次，没有后续动作、反应或状态变化 | `C` |
| 某泛称太医、侍卫、百姓在不同位置重复出现，但没有证据证明是同一具体人物 | `C` 或拆成不同 C/模板行 |

背景、群体、泛称角色不因重复出现自动升级。它们可以作为模板或群体资产处理，因为生产上没有单人脸模连续要求。

## 评级输出字段

`asset-rating-result` 中的人物评级项只使用以下字段：

| 字段 | 是否必填 | 允许值/规则 | 含义 |
|---|---:|---|---|
| `base_asset_id` | 是 | 稳定人物资产 ID | 被评级的人物资产 |
| `asset_type` | 是 | `character` | 资产类型 |
| `asset_name` | 是 | 规范化人物名 | 后续注册和锚定引用名 |
| `priority_level` | 是 | `S/A/B/C` | 脸模连续性等级 |
| `same_asset_evidence` | 是 | `strong/medium/weak/conflict` | 判断是否同一人的证据强度 |
| `episode_span` | 是 | 集数列表或单集 | 人物确认出现范围 |
| `scene_ids` | 是 | 场次 ID 列表 | 评级依据场次 |
| `rating_reason` | 是 | 文本 | 为什么评为该等级 |
| `source_evidence` | 是 | 剧本证据 | 支持评级的证据 |
| `rating_uncertainty_note` | 条件必填 | 文本或空 | 同一人物证据不足、冲突或降级原因 |

## 边界

- 不拆分人物状态。
- 不判断 `task_type`。
- 不设置 `anchor_task_id`。
- 不设置 `review_status`。
- 不生成提示词。
- 不生成分集映射。

只使用以上列举过的字段和职责。
