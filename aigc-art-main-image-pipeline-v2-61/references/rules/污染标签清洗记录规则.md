# 污染标签清洗记录规则

## 规则目的

本文件只用于约束 `asset-extraction.md` 输出 `polluted-label-cleanup-ledger`。

本规则解决的唯一问题是：

> 被人物候选准入门挡下的污染标签，如何留下可审计清洗记录，而不污染 `asset-candidate-list` 和 `character-disposition-ledger`？

本规则不替代 `资产抽取边界规则.md` 或 `人物候选去向闭环规则.md`。真实人物候选仍由 `asset-candidate-list` 承接；真实人物候选去向由 `character-disposition-ledger` 记录；污染标签清洗记录只记录被归一、拆分、保留为证据或阻断复核的过程。

## 输入依据

必须读取前置层级已经产出的结构化结果：

- `character-evidence-ledger`
- `scene-coverage-table`
- 同层生成的 `asset-candidate-list`

可读取 `source_evidence` 原文证据定位污染标签。不得跳过 `character-evidence-ledger` 或 `scene-coverage-table` 重新切分场次、重新评级、重新注册资产或生成任务。

## 输出结果

输出结果固定命名为 `polluted-label-cleanup-ledger`。

`polluted-label-cleanup-ledger` 是污染标签清洗记录，不是人物候选去向台账，不是中台生产任务表，不参与人物候选数量、S/A/B/C 数量或生产任务数量统计。

## 记录范围

以下内容只能进入 `polluted-label-cleanup-ledger`，不得进入 `asset-candidate-list` 或 `character-disposition-ledger`：

- 台词前缀污染形成的人物短语，例如 `皇帝点头`、`棠宁喃喃`、`皇后拍桌`。
- 人物名后拼接动作、情绪、语气、声音或瞬时表情形成的短语，例如 `棠宁大惊`、`萧厌低声`。
- 多人组合或动作组合短语，例如 `两人一起`、`二夫人三夫人一起`。
- 未完成上下文消歧的可转移称谓，例如未绑定具体人物和时间范围的 `四夫人`、`夫人`、`主子`。

如果污染短语已经被可靠归一为真实主体，则真实主体可以作为人物候选进入 `asset-candidate-list`，污染短语本身仍只能作为 `character-evidence-ledger.raw_label`、清洗记录或原文证据存在。

## 必填字段

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `cleanup_id` | 是 | 稳定清洗记录 ID。 |
| `raw_label` | 是 | 原始污染标签，只允许在本字段或 `source_evidence` 中保留污染文本。 |
| `normalized_subject` | 条件必填 | 能可靠识别主体时填写规范主体名；无法识别时留空并写明原因。 |
| `cleanup_type` | 是 | 污染类型，只允许本规则定义的枚举。 |
| `source_scene_ids` | 是 | 来源场次 ID 列表。 |
| `source_evidence_ids` | 是 | 来源 `character-evidence-ledger.evidence_id` 列表；无法回指时必须 `blocking_review`。 |
| `source_evidence` | 是 | 支持清洗判断的原文证据。 |
| `handling_result` | 是 | 清洗结果，只允许本规则定义的枚举。 |
| `target_candidate_id` | 条件必填 | `normalized_to_subject` 或 `split_to_subjects` 且主体进入候选池时填写，必须能在 `asset-candidate-list` 找到。 |
| `review_note` | 条件必填 | 证据不足、阻断复核、未消歧或无法识别主体时必填。 |

## 枚举值

`cleanup_type` 只允许以下取值：

| `cleanup_type` | 含义 |
|---|---|
| `dialogue_prefix_modifier` | 台词前缀或说话主体后拼接动作、语气、表情。 |
| `action_emotion_speech` | 人物名后拼接动作、情绪、语气、声音或瞬时表情。 |
| `combination` | 多人组合、数量词组合或动作组合。 |
| `transferable_title` | 未消歧的可转移称谓。 |
| `unresolved_label` | 无法可靠识别主体的污染标签。 |

`handling_result` 只允许以下取值：

| `handling_result` | 含义 |
|---|---|
| `normalized_to_subject` | 污染标签已归一到一个真实主体。 |
| `split_to_subjects` | 组合标签已拆成多个真实主体。 |
| `evidence_only` | 不进入候选池，只保留为原文证据或场面证据。 |
| `blocking_review` | 证据不足或冲突，需要阻断复核。 |

不得新增其他枚举。不得把 `handling_result` 当作 `final_disposition`、`task_type` 或 `review_status` 使用。

## 处理规则

- 能识别单个真实主体时，`raw_label` 写污染文本，`normalized_subject` 写主体名，`handling_result` 写 `normalized_to_subject`。
- 能识别多个真实主体时，`handling_result` 写 `split_to_subjects`，`normalized_subject` 可写拆分后的主体名列表，`target_candidate_id` 必须能对应到候选主体。
- 不能识别主体但不影响资产判断时，`handling_result` 写 `evidence_only`，不得生成候选人物。
- 污染标签影响候选、评级、注册或任务判断且证据不足时，`handling_result` 写 `blocking_review`，并在 `review_note` 写明需要回到哪个上游层级复核。
- 可转移称谓只有在绑定具体人物和时间范围后，才能作为主体称谓证据；未绑定时不得生成或合并人物候选。

## 审计交接要求

`polluted-label-cleanup-ledger` 必须支持下游审计：

- 每条记录字段完整，枚举合法。
- `target_candidate_id` 如存在，必须能在 `asset-candidate-list` 找到。
- `raw_label` 不得进入 `asset-candidate-list.raw_name`、`character-disposition-ledger.raw_name`、`normalized_name`、`asset_name`、`state_name`、`task_id` 对应名称或分集映射名称。
- `blocking_review` 必须由 `production-audit.md` 标为 `blocking`。
- 清洗记录不得计入人物候选、S/A/B/C 评级或生产任务统计。

## 边界

本规则只定义污染标签清洗记录：

- 不重新切分场次。
- 不判断同一人物证据。
- 不评级 S/A/B/C。
- 不注册资产。
- 不生成候选去向。
- 不生成任务。
- 不建立脸模锚点。
- 不生成提示词。
- 不定义全局生产任务字段。

只使用以上列举过的字段、枚举和职责。
