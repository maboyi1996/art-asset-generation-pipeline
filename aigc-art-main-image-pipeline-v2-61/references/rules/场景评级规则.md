# 场景评级规则

## 规则目的

本文件只用于给场景 base asset 评定制作连续性等级 `S/A/B/C`。

场景评级是**空间连续性与生产复用等级**，不是戏剧强度评分。核心问题是：

> 这个场景/空间设计需要在多大范围内保持一致？

本文件只定义同一场景证据、评级顺序、等级含义和评级输出字段。

本文件不定义状态拆分、任务类型、分集映射、提示词模板或最终输出表格式。

## 输入依据

场景评级必须基于前置层级已经产出的结构化结果：

- `scene-evidence-ledger`
- `asset-candidate-list`
- `state-variant-result`
- `scene-coverage-table`
- 剧本原文 `source_evidence/source_locator`

`scene-evidence-ledger` 是场景评级主输入；`scene-coverage-table` 用于上下文；剧本原文定位只用于核验同一空间证据；`asset-candidate-list` 是候选承接结果，不能作为唯一事实源。禁止只根据地点名称出现频次评级。

## 评级决策顺序

场景评级必须按以下顺序判断：

0. 对照 `scene-evidence-ledger` 检查可见空间事实是否被候选承接；漏承接时阻断并指回 `scene-evidence-ledger.md` 或 `asset-extraction.md`，不得静默补候选。
1. 是否能明确认定多次出现的是**同一个地点 / 同一个空间资产**？
2. 如果不能，默认不是同一个资产，不累计出现次数，不升级。
3. 如果能，判断是否跨多集出现。
4. 明确同一空间跨多集出现：评为 `S` 或 `A`。
5. 明确同一空间只在单集多场出现，或单集内具有强设计需求：评为 `B`。
6. 单次、泛化、过渡、提及、被上级场景覆盖、无法确认同一空间：评为 `C`。

## 同一场景证据门

只有同一场景证据成立，才能累计出现次数。

可以作为强证据的情况：

- 稳定命名地点：`宋府正厅`、`皇后寝宫`、`大理寺审堂`。
- 剧本明确写明回到同一地点。
- 同一归属者、家族或机构，并且同一房间功能明确。
- 有独特牌匾、布局、地标、材质、破损、事件布置。
- 单集内上下场直接承接，能明确判断为空间连续。

证据不足的情况：

- 泛化地点：`街道`、`巷口`、`房间`、`门口`、`宫门`、`走廊`、`院子`。
- 只有相同空间类型。
- 只有相同时间段。
- 重复出现的泛化过渡空间。
- 相似宫殿、办公间、道路描述，但没有证据证明是同一个具体空间。

证据不足时，默认不是同一个场景资产。

## 等级定义

| 等级 | 定义 | 连续性范围 | 评级含义 |
|---|---|---|---|
| `S` | 项目级核心地点。同一地点跨多集出现，并定义家族、机构、阵营、阶层世界或长期故事空间。 | 全项目 | 后续作为全局场景资产复用。 |
| `A` | 项目级复用地点。同一地点跨多集出现，但频次、世界观依赖或主线权重低于 S。 | 全项目或主要段落 | 后续作为全局场景资产复用。 |
| `B` | 单集级场景。同一地点在单集多场出现，或单集事件场景有明确设计需求。 | 单集或短段落 | 后续作为本集/短段落场景资产。 |
| `C` | 泛化、过渡、提及、被覆盖、或无法确认同一空间的地点。 | 不固定 | 通常不需要独立空间连续。 |

## S 与 A 的区别

使用 `S` 的情况：

- 全项目高频出现。
- 长期承载主线、家族、机构或世界观。
- 有多个状态或大量生产复用。
- 是项目核心视觉空间。

使用 `A` 的情况：

- 同一地点跨多集出现。
- 频次、状态复杂度、项目视觉身份权重低于 `S`。

不得因为泛化地点出现次数多就升级，例如走廊、道路、房间、门口、院子。

## B 级规则

满足以下情况时评为 `B`：

| 情况 | 评级 |
|---|---|
| 某审堂只在一集内多场出现 | `B` |
| 临时牢房/审讯室支撑一集主要事件 | `B` |
| 单集宴席布置具有明确视觉设计需求 | `B` |
| 单集战场、仪式、婚礼、葬礼、审判场景需要主参考图 | `B` |

如果只是被上级场景覆盖的泛化房间，保持 `C` 或在评级原因中说明父级覆盖关系。

## C 级规则

没有独立资产连续需求，或同一空间证据不足时，评为 `C`。

| 情况 | 评级/处理 |
|---|---|
| 泛化道路、门口、走廊、角落、空房间 | `C`，说明泛化或父级覆盖 |
| 只在对白中提及的地点 | `C`，写明无视觉生产需求 |
| 无设计需求的短过渡背景 | `C` |
| 多次出现“宫门口”，但无法证明是同一个具体宫门资产 | `C` 或拆成本地场景行，不升级 |

## 评级输出字段

`asset-rating-result` 中的场景评级项只使用以下字段：

| 字段 | 是否必填 | 允许值/规则 | 含义 |
|---|---:|---|---|
| `base_asset_id` | 是 | 稳定场景资产 ID | 被评级的场景资产 |
| `asset_type` | 是 | `scene` | 资产类型 |
| `asset_name` | 是 | 规范化场景名 | 后续注册和映射引用名 |
| `priority_level` | 是 | `S/A/B/C` | 空间连续性/复用等级 |
| `same_asset_evidence` | 是 | `strong/medium/weak/conflict` | 判断是否同一空间资产的证据强度 |
| `episode_span` | 是 | 集数列表或单集 | 场景确认出现范围 |
| `scene_ids` | 是 | 场次 ID 列表 | 评级依据场次 |
| `rating_evidence_ids` | 是 | `scene-evidence-ledger.evidence_id` 列表 | 支持本次评级的结构化证据回指 |
| `rating_reason` | 是 | 文本 | 为什么评为该等级 |
| `source_evidence` | 是 | 剧本证据 | 支持评级的证据 |
| `rating_uncertainty_note` | 条件必填 | 文本或空 | 同一空间证据不足、冲突或降级原因 |

`rating_evidence_ids` 不得为空；不得只写自然语言 `source_evidence` 而缺少 evidence ledger 回指。若评级所需证据尚未进入 `scene-evidence-ledger`，必须阻断回 `scene-evidence-ledger.md` 或 `asset-extraction.md`。

## 边界

- 不拆分场景状态。
- 不判断 `task_type`。
- 不设置 `review_status`。
- 不生成提示词。
- 不生成分集映射。
- 不重新扫 PDF。
- 不重新切分集场。
- 不静默补写候选；候选或证据账本漏项必须阻断回上游。

只使用以上列举过的字段和职责。
