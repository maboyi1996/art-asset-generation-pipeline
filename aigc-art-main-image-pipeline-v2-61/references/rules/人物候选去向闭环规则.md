# 人物候选去向闭环规则

## 规则目的

本文件只用于约束 `candidate-disposition.md` 为每一个人物候选输出最终去向台账，并为泛称/群体人物输出模板覆盖台账。

本规则解决的唯一问题是：

> 每一个进入 `asset-candidate-list` 的人物候选，最终是被生产、作为状态任务、模板化、别名合并、分组、排除、标记为非视觉候选，还是交给人工审核？

本规则防止 `character-anchor-result` 被误当作完整人物清单，防止 B/C 人物因为不需要项目级脸模锚点而在后续输出中静默消失。

本规则不替代 `全局资产任务生成规则.md`。可生产人物必须由本台账给出生产类去向，再由 `task-manifest-generation.md` 生成真实 `global-task-draft` 任务行。

本规则也不记录污染标签清洗过程。被人物候选准入门挡下的 `皇帝点头`、`棠宁喃喃`、`皇后拍桌`、`二夫人三夫人一起` 等污染标签必须进入 `polluted-label-cleanup-ledger`，不得作为伪人物候选写入本台账。

## 输入依据

必须读取前置层级已经产出的结构化结果：

- `asset-candidate-list`
- `asset-rating-result`
- `global-asset-registry`
- `alias-resolution-map`
- `polluted-label-cleanup-ledger`

可读取 `state-variant-result` 中的状态 ID 和状态证据，但不得重新拆状态。

不得跳过前置结果重新抽取人物、重新切分场次、重新评级或重新注册资产。

`character-disposition-ledger` 的输入范围只限 `asset-candidate-list` 中 `asset_type: character` 的人物候选。未进入 `asset-candidate-list` 的污染标签、动作短语、情绪短语、语气短语、组合短语和未消歧称谓不属于本台账输入。

## 输出结果

输出结果固定命名为：

- `character-disposition-ledger`
- `generic-template-coverage-ledger`

`character-disposition-ledger` 是人物候选去向台账，不是中台生产任务表。它被 `production-audit.md` 审计，并由 `output-contracts.md` 写入机器长表的独立 `人物候选去向审计表`。

`generic-template-coverage-ledger` 记录泛称、群体、背景人物被模板任务、集合任务、分组或可见排除覆盖的关系，防止泛称人物既不生产也不说明去向。

本台账必须保持人物候选干净：`raw_name` 和 `normalized_name` 不得包含动作、情绪、语气、声音、瞬时表情、组合词或未消歧可转移称谓。污染原文只能出现在 `disposition_evidence` 的原文证据中，或出现在 `polluted-label-cleanup-ledger.raw_label`。

## 去向枚举

`final_disposition` 只允许以下取值：

| `final_disposition` | 含义 | 必填关联字段 |
|---|---|---|
| `production_task` | 应形成独立生产任务。 | `task_id` |
| `state_task` | 应作为某人物状态任务。 | `task_id` |
| `template_task` | 应形成模板类任务，例如泛称群体模板。 | `task_id` |
| `alias_to` | 作为别名合并到另一个人物。 | `alias_to` |
| `grouped_into` | 归入群体、模板或集合资产。 | `grouped_into` |
| `excluded` | 明确排除，不生产。 | `excluded_reason` |
| `not_visual_candidate` | 非视觉候选，不进入生产。 | `excluded_reason` |
| `manual_review_required` | 证据不足、冲突且暂时无法形成合法生产任务，需要人工决定。 | `disposition_uncertainty_note` |

不得新增其他去向枚举。不得把 `final_disposition` 当作 `task_type` 使用。

`production_task`、`state_task`、`template_task` 的 `task_id` 是稳定计划任务 ID；`task-manifest-generation.md` 必须在 `global-task-draft` 中生成同 ID 任务行。生产审计负责检查该计划 ID 是否被真实任务表承接。

## `character-disposition-ledger` 必填字段

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `candidate_id` | 是 | 继承 `asset-candidate-list`，必须稳定可追踪。 |
| `raw_name` | 是 | 原文人物名或称呼。 |
| `normalized_name` | 是 | 候选或注册后的规范名；证据不足时不要过度合并。 |
| `presence_type` | 是 | 继承场次覆盖或候选信息，例如 `physical_visible`、`memory_visible`、`portrait_visible`、`corpse_visible`、`voice_only`、`mentioned_only`、`memorial_reference`、`rumor_mentioned`。 |
| `scene_ids` | 是 | 来源场次 ID 列表。 |
| `source_evidence_ids` | 是 | 继承 `asset-candidate-list.source_evidence_ids`，用于回指评级前 evidence ledger。 |
| `candidate_scope` | 是 | 继承候选范围，例如 `specific`、`named_or_stable`、`generic`。 |
| `base_asset_id` | 条件必填 | 已注册为人物资产时必填；未注册时必须说明去向原因。 |
| `priority_level` | 条件必填 | 继承 `asset-rating-result`，不得在本规则重新评级。 |
| `same_asset_evidence` | 条件必填 | 继承或引用同一人物证据。 |
| `final_disposition` | 是 | 只允许本规则定义的枚举。 |
| `task_id` | 条件必填 | `production_task`、`state_task`、`template_task` 时必填，且必须被 `task-manifest-generation.md` 承接为真实任务行。 |
| `alias_to` | 条件必填 | `alias_to` 时必填，且必须指向已登记人物或候选。 |
| `grouped_into` | 条件必填 | `grouped_into` 时必填，且必须指向已登记群体、模板计划任务、集合任务或可见排除记录。 |
| `excluded_reason` | 条件必填 | `excluded` 或 `not_visual_candidate` 时必填。 |
| `disposition_evidence` | 是 | 支持该去向的场次证据、注册证据或计划任务证据。 |
| `disposition_uncertainty_note` | 条件必填 | 证据不足、冲突、泛称合并不确定或人工审核时必填。 |

## `generic-template-coverage-ledger` 字段

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `coverage_id` | 是 | 稳定覆盖记录 ID。 |
| `source_candidate_ids` | 是 | 被模板、分组或排除覆盖的泛称/群体候选。 |
| `generic_label` | 是 | 泛称或群体标签，例如 `侍卫`、`下人`、`宾客`。 |
| `coverage_result` | 是 | `template_task`、`grouped_into`、`excluded`、`not_visual_candidate`、`manual_review_required`。 |
| `task_id` | 条件必填 | `template_task` 时必填，并由 `task-manifest-generation.md` 承接。 |
| `grouped_into` | 条件必填 | `grouped_into` 时必填。 |
| `excluded_reason` | 条件必填 | 排除或非视觉时必填。 |
| `coverage_evidence` | 是 | 说明为何模板化、分组、排除或送审。 |
| `coverage_uncertainty_note` | 条件必填 | 证据不足或冲突时必填。 |

## 去向判断规则

| 人物候选情况 | 合法去向 |
|---|---|
| S/A 人物基础形象 | `production_task`，并在后续 `character-anchor-result` 建立项目级锚点。 |
| S/A 人物状态资产 | `state_task`，状态任务应引用对应锚点。 |
| B 人物单集连续角色 | `production_task`；不要求项目级锚点，但必须计划 `text_to_image` 任务。证据冲突时仍应计划 `review_status: manual_review_required` 的任务行。 |
| C 人物有明确单独视觉生产价值 | `production_task` 或 `template_task`；必须计划 `text_to_image` 任务。证据冲突时仍应计划 `review_status: manual_review_required` 的任务行。 |
| C 泛称群体或功能人物 | 默认 `template_task`、`grouped_into`、`excluded` 或 `not_visual_candidate`；并写入 `generic-template-coverage-ledger`。 |
| 只被提及且没有视觉生产需求 | `not_visual_candidate` 或 `excluded`。 |
| VO、旁白、台词提及、牌位坟墓棺材等纪念物指代 | 不作为可见真人生产任务；如无人物形象生产需求，使用 `not_visual_candidate` 或 `excluded`，并说明可转为道具/场景证据。 |
| 别名、称谓、同一人物重复候选 | `alias_to`，必须写明合并证据。 |
| 证据不足且无法形成合法任务 | `manual_review_required`，必须写明缺失证据；如果仍可形成任务，则使用对应任务去向并要求任务行设置 `review_status: manual_review_required`。 |

B/C 人物不得因为“不需要项目级锚点”而没有去向。

`manual_review_required` 不是生产替代品。只要人物仍被判定为可生产资产，就必须规划任务 ID，并由 `task-manifest-generation.md` 在 `global-task-draft` 中生成任务行。

## 泛称人物处理

- 泛称人物可以被模板化、分组、排除或送审，但不得静默丢弃。
- 多场出现相同泛称时，不得仅因名称相同就合并为同一个具体人物。
- 如果泛称人物被归入群体模板，必须填写 `generic-template-coverage-ledger`。
- 如果泛称人物被排除，必须填写 `excluded_reason`，说明其只作为场面背景、无独立视觉生产需求或已由场景/群演模板覆盖。
- 如果泛称人物被生成独立 `production_task`，必须写明为什么它不是模板、分组或排除；否则审计应视为泛称任务爆炸风险。
- 重复出现的泛称标签不得仅因跨集或跨场拆成多个独立单人脸模任务。

## 审计交接要求

`character-disposition-ledger` 和 `generic-template-coverage-ledger` 必须支持下游审计：

- `asset-candidate-list` 中每个 `asset_type: character` 候选都有且只有至少一条最终去向记录。
- 每条去向记录必须保留 `source_evidence_ids`，不得只用自然语言 `disposition_evidence` 替代 evidence ledger 回指。
- 每条记录的 `final_disposition` 合法。
- 指向计划任务的记录必须能在 `global-task-draft` 找到同 ID `task_id`。
- 指向别名、分组或模板覆盖的记录必须能找到目标。
- B/C 可生产人物的 `production_task`、`state_task` 或 `template_task` 必须能在任务草稿中找到真实 `task_id`。
- 污染标签清洗记录不得作为人物候选去向记录出现。
- `manual_review_required` 不得作为可生产人物无任务时的最终替代去向。
- 排除、非视觉候选、人工审核必须有原因。
- B/C 人物缺去向必须由 `production-audit.md` 标为 `blocking`。

## 边界

本规则只定义人物候选去向闭环：

- 不重新抽取人物。
- 不重新切分场次。
- 不重新判断同一人物证据。
- 不重新评级 S/A/B/C。
- 不拆分状态。
- 不生成真实任务表。
- 不建立脸模锚点。
- 不判断提示词内容。
- 不定义最终 Markdown 排版。
- 不新增中台生产任务字段。
- 不记录污染标签清洗过程。

只使用以上列举过的字段、枚举和职责。
