# 生产审计规则

## 规则目的

本文件只用于 `production-audit.md` 的生产前一致性审计。

生产审计的核心问题是：

> 前序结构化结果是否完整、一致、可被提示词生成层安全读取？

本文件只定义审计对象、审计问题类型、严重程度和 `production-audit-result` 输出字段。

本文件不重新抽取资产、不重新拆状态、不重新评级、不重写提示词、不直接修复注册表、候选去向、任务草稿或分集映射。

## 审计对象

| 审计对象 | 检查内容 |
|---|---|
| `scene-coverage-table` | 场次事实是否被候选资产、清洗记录、映射或可见排除记录覆盖。 |
| `character-evidence-ledger` | 人物证据是否被候选、污染清洗、评级、去向、任务、模板、排除或阻断复核承接。 |
| `scene-evidence-ledger` | 场景证据是否被候选、评级、注册、任务、映射、排除或阻断复核承接。 |
| `prop-evidence-ledger` | 道具证据是否被候选、评级、注册、任务、映射、排除或阻断复核承接。 |
| `asset-candidate-list` | 候选资产是否都有注册、合并、排除、候选去向或审核去向。 |
| `polluted-label-cleanup-ledger` | 污染标签是否只在清洗记录或原文证据中出现，字段是否合法，目标候选是否可追踪。 |
| `state-variant-result` | 状态是否挂到正确 base asset，是否存在状态孤儿。 |
| `state-clue-disposition-ledger` | 每条状态线索是否有采用、复用、排除、证据保留或送审去向。 |
| `asset-rating-result` | 每个 base asset 是否有唯一 `priority_level`，状态是否继承等级。 |
| `global-asset-registry` | 资产 ID、状态 ID 是否稳定且能回溯到候选和状态结果。 |
| `alias-resolution-map` | 别名合并是否有证据；未消歧称谓是否没有被强行合并。 |
| `character-disposition-ledger` | 每个人物候选是否有最终去向，B/C 是否未被静默丢失，去向是否能被任务或可见排除承接。 |
| `generic-template-coverage-ledger` | 泛称、群体、背景人物是否有模板、分组、排除、非视觉或送审覆盖。 |
| `global-task-draft` | 应生产资产是否都有任务行，`task_id`、`task_type`、`review_status`、`anchor_task_id` 是否完整合法。 |
| `episode-usage-map-draft` | 分集映射是否能引用到存在的 `task_id`，或引用明确排除/非视觉/模板覆盖记录。 |
| `character-anchor-result` | S/A 人物是否有锚点任务，状态图是否依赖锚点；不得当作完整人物清单。 |

## 问题类型

| 问题类型 | 审计动作 |
|---|---|
| 缺必要字段 | 输出审计问题，指向负责字段的上游层级。 |
| 缺 evidence 回指字段 | 输出阻断问题；候选、评级、注册、人物去向或污染清洗记录缺少对应 evidence ID 时，指回字段拥有层。 |
| 场次事实无去向 | 输出覆盖问题，要求回到 `asset-extraction.md`、`state-variant.md`、`candidate-disposition.md` 或 `episode-usage-map.md`。 |
| 污染清洗记录缺失或断链 | 输出阻断问题；要求回到 `asset-extraction.md`。 |
| 污染人物名进入结构化名称 | 输出阻断问题；按污染所在结构指回对应层级。 |
| 状态线索无去向 | 输出阻断问题；要求回到 `state-variant.md`。 |
| 状态孤儿 | 输出阻断问题；要求回到 `state-variant.md` 或 `asset-registry.md`。 |
| 候选资产无注册或去向 | 输出阻断问题；注册问题回 `asset-registry.md`，人物去向问题回 `candidate-disposition.md`。 |
| 人物候选无最终去向 | 输出阻断问题，要求回到 `candidate-disposition.md`。 |
| 泛称/群体无覆盖 | 输出阻断问题，要求回到 `candidate-disposition.md`。 |
| 应生产资产缺任务 | 输出阻断问题；要求回到 `task-manifest-generation.md`。 |
| `task_id` 断链 | 输出阻断问题；候选去向计划 ID 未被物化时回 `task-manifest-generation.md`，映射引用缺失时回 `episode-usage-map.md`。 |
| B/C 可生产人物缺 `text_to_image` 任务 | 输出阻断问题；B/C 可生产人物必须有生产任务，不能只存在于人物候选去向台账。 |
| B/C 人物无闭环 | 输出阻断问题；B/C 人物必须有真实生产任务、状态任务、模板任务、别名、有效分组、排除或非视觉去向。 |
| 锚点结果误作完整人物清单 | 输出阻断问题，要求回到 `output-contracts.md`，不能用 `character-anchor-result` 代替人物候选台账。 |
| 人工审核状态替代任务 | 输出阻断问题；`manual_review_required` 只能作为任务行 `review_status`，不能替代应生产资产的 `task_id`。 |
| 空分组替代任务 | 输出阻断问题；`grouped_into` 必须指向真实模板任务、集合任务、别名目标或可见排除记录。 |
| 泛称人物粗暴合并 | 输出警告或阻断问题；缺少同一人物证据时要求回到 `asset-registry.md` 或 `candidate-disposition.md` 拆分、分组、模板化或排除。 |
| 分集映射缺引用 | 输出映射问题，要求回到 `episode-usage-map.md`。 |
| 审核状态与任务字段冲突 | 输出字段问题，要求回到 `task-manifest-generation.md` 或 `任务类型字段审核规则.md`。 |
| 疑似场次标题未解析 | 输出阻断问题；要求回到 `script-intake.md` 修正场次边界，例如 `65-2（闪回）...` 不得残留在上一场文本中。 |
| 证据账本漏项 | 输出阻断问题；要求回到对应 evidence layer 或 `scene-coverage.md`。 |
| 证据账本事实未被候选承接 | 输出阻断问题；要求回到 `asset-extraction.md`，不能靠评级、注册或任务层补候选。 |
| 人物状态主体不明 | 输出阻断问题；状态不能只由场级关键词触发并分配给多个人物。 |
| 非法人物状态 | 输出阻断问题；纯动作、纯情绪、语气、声音、短暂表情不得成为状态任务。 |
| 泛称人物任务爆炸 | 输出阻断问题；泛称、群体或背景人物不得仅因跨场/跨集重复出现而生成大量独立单人脸模任务。 |
| A/S 评级缺少可见脸模证据 | 输出阻断问题；A/S 不能只由 VO、提及、牌位、坟墓、棺材或纪念物支撑。 |

生产审计只能给出 `required_action`，不得在本层直接补字段、补注册、补人物任务、改评级、改状态或重写提示词。

## 非统计型阻断原则

不得因为某项目集数长、B/C 数量低、S/A/B/C 汇总比例异常或模板任务数量看起来偏少，直接输出 `blocking`。

只有出现以下证据级问题时才允许阻断：

- 可见事实没有进入候选、清洗、状态线索、去向、任务、映射或可见排除路径。
- evidence ledger 中的可见事实没有进入候选、评级、清洗、去向、任务、映射或可见排除路径。
- 已进入候选的人物没有最终去向。
- 生产类去向没有被 `global-task-draft` 承接。
- 分集映射引用不存在的任务或排除记录。
- 污染标签进入结构化名称。
- 状态主体不明却生成状态任务。
- A/S、B/C 硬分流所需的同一具体人物证据链断裂。

长剧数量异常只能作为 `warning` 或审计说明，用来提示人工抽查评级和候选覆盖，不得替代逐事实闭环审计。

## 严重程度

| `severity` | 含义 | 是否可进入提示词生成 |
|---|---|---:|
| `blocking` | 会导致任务断链、锚点缺失、字段无法读取或资产去向缺失。 | 否 |
| `warning` | 可以生成，但需要人工确认或后续复核。 | 是 |
| `info` | 不影响生产，只用于说明。 | 是 |

## 输出要求

`production-audit-result` 至少包含：

- `audit_status`
- `issue_id`
- `issue_type`
- `affected_layer`
- `affected_id`
- `severity`
- `required_action`
- `suggested_review_status_update`
- `review_reason`

审计结果必须被 `prompt-generation.md`、`output-contracts.md` 和 `final-quality.md` 读取。

## 任务完整性审计

生产审计必须交叉检查：

- S/A/B/C 已注册且应生产的角色、场景、道具资产是否都有 `global-task-draft` 行。
- `character-evidence-ledger`、`scene-evidence-ledger`、`prop-evidence-ledger` 中的可见事实是否被 `asset-candidate-list`、污染清洗、评级、注册、去向、任务、映射或排除记录闭环承接。
- `asset-rating-result.source_evidence` 是否引用对应 evidence ledger 或其 `source_locator` 原文证据，不能只引用资产名或统计频次。
- `asset-candidate-list.source_evidence_ids`、`asset-rating-result.rating_evidence_ids`、`global-asset-registry.source_evidence_ids/rating_evidence_ids`、`character-disposition-ledger.source_evidence_ids`、`polluted-label-cleanup-ledger.source_evidence_ids` 是否存在且能回指真实 evidence ledger 行。
- B 级人物基础形象是否都有 `text_to_image` 任务和 `episode_id`。
- C 级可视单人或模板人物如果保留为资产，是否有 `text_to_image` 任务或合法可见排除记录。
- C 级人物是否符合硬分流：不能确认同一具体人物；或明确同一具体人物只在单集出现，且全部可见戏份能被证明处于同一个最长 15 秒单片段内。
- `production_task`、`state_task`、`template_task` 去向是否都能在 `global-task-draft` 找到对应 `task_id`。
- `manual_review_required` 是否只出现在任务行 `review_status` 或不可形成任务的去向记录中。
- `grouped_into` 是否指向真实模板任务、集合任务、别名目标或可见排除记录。
- `script-intake-result` 或 `scene-coverage-table` 中是否存在疑似未解析场次标题；存在时不得进入提示词生成。
- `scene-coverage-table.character_mentions`、`asset-candidate-list`、`global-asset-registry`、`alias-resolution-map`、`character-disposition-ledger`、`global-task-draft`、`episode-usage-map-draft` 或状态任务中是否存在污染名。
- 污染短语是否只保留在 evidence ledger raw fields、`source_evidence` 原文证据和 `polluted-label-cleanup-ledger.raw_label` 中。
- `polluted-label-cleanup-ledger` 是否包含必填字段，`cleanup_type` 和 `handling_result` 是否使用合法枚举，`target_candidate_id` 是否能在 `asset-candidate-list` 中找到。
- `polluted-label-cleanup-ledger` 是否被错误计入人物候选数量、S/A/B/C 数量、生产任务数量或 `review_status` 统计。
- `四夫人`、`夫人`、`主子`、`管事` 等可转移称谓是否已经绑定具体人物和时间范围。
- 人物状态任务是否能回溯到明确主体证据；如果同一场的状态关键词被分配给多个无主体证据人物，必须阻断。
- A/S 人物的跨集依据是否包含可见脸模生产出场；如果只由 `voice_only`、`mentioned_only`、`memorial_reference` 或等价证据支撑，必须阻断并指回评级层。
- C 泛称或群体人物是否被大量拆成独立单人 `production_task`；如果没有模板差异、场面差异或同一具体人物证据，必须阻断并指回去向或任务层模板化、分组或排除。

## 层级指向

- 疑似场次标题未解析：`script-intake.md`。
- 污染源位于 `character_mentions`：`scene-coverage.md`。
- 人物证据账本漏项或人物可见性记录错误：`character-evidence-ledger.md`。
- 场景证据账本漏项或空间证据记录错误：`scene-evidence-ledger.md`。
- 道具证据账本漏项或物件证据记录错误：`prop-evidence-ledger.md`。
- 污染源位于 `asset-candidate-list` 或清洗记录缺失：`asset-extraction.md`。
- 状态主体不明、非法状态、状态线索无去向：`state-variant.md`。
- A/S 评级缺少可见脸模证据，或 S/A/B/C 硬分流证据断裂：`asset-rating.md`。
- 注册表、别名、状态 ID 断链：`asset-registry.md`。
- 人物候选去向、泛称覆盖、空分组断链：`candidate-disposition.md`。
- 应生产资产缺任务、候选去向计划 ID 未被物化：`task-manifest-generation.md`。
- 分集映射引用断链：`episode-usage-map.md`。
- 锚点依赖断链：`character-anchor.md`。
- 机器长表或人审短表使用错误来源：`output-contracts.md`。

## 边界

- 不重新抽取资产。
- 不重新拆状态。
- 不重新评级。
- 不建立锚点。
- 不生成提示词。
- 不修改最终 Markdown 输出。

只使用以上列举过的字段和职责。
