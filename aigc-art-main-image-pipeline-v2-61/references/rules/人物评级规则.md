# 人物评级规则

## 规则目的

本文件只用于给人物 base asset 评定制作连续性等级 `S/A/B/C`。

人物评级不是剧情重要性评分，而是脸模连续性等级。核心问题是：

> 这个人物的脸需要在多大范围内保持一致？

本文件只定义人物候选准入门、同一人物证据门、可见出场计数口径、硬分流算法、等级定义和评级输出字段。

本文件不定义状态拆分、任务类型、锚点流程、分集映射、提示词模板或最终输出表格式。

## 输入依据

人物评级必须基于前置层级已经产出的结构化结果：

- `character-evidence-ledger`
- `asset-candidate-list`
- `state-variant-result`
- `scene-coverage-table`
- 剧本原文 `source_evidence/source_locator`

`character-evidence-ledger` 是人物评级主输入；`scene-coverage-table` 用于上下文；剧本原文定位只用于核验证据；`asset-candidate-list` 是候选承接结果，不能作为唯一事实源。禁止只根据名称出现频次评级。

## 人物候选准入门

正式人物评级项只能对应已经通过 `资产抽取边界规则.md` 准入门的真实主体候选；但评级层必须读取 `character-evidence-ledger` 检查是否存在可见人物事实未被候选承接。发现漏承接时，不得静默补候选或降级兜底，必须阻断并指回 `character-evidence-ledger.md` 或 `asset-extraction.md`。

以下内容不得进入人物评级：

- 污染名，例如 `皇帝点头`、`棠宁喃喃`、`皇后拍桌`。
- 动作、情绪、语气、声音或瞬时表情短语。
- 组合名，例如 `两人一起`、`二夫人三夫人`、`二夫人三夫人一起`。
- 未消歧的可转移称谓，例如未绑定具体人物和时间范围的 `四夫人`、`夫人`、`主子`。
- 只存在于证据原文中的修饰词。

如果以上内容已经进入候选池，必须退回 `asset-extraction.md` 或 `scene-coverage.md` 归一、拆分或排除，不得在评级层用 `B/C` 兜底。污染原文只允许出现在 `character-evidence-ledger.raw_label`、`polluted-label-cleanup-ledger.raw_label` 或 `source_evidence`。

## 同一人物证据门

只有同一人物证据成立，才能累计出现范围并进入硬分流。

可以作为同一具体人物证据的情况：

- 稳定姓名、封号、职务加唯一归属，例如 `宋夫人`、`陆皇后`。
- 剧本明确写明同一人、仍是他/她、上次那人等连续指代。
- 与 S/A 人物存在稳定关系绑定，例如同一个母亲、丈夫、随从、贴身侍女。
- 有连续事件链、伤病状态、身份关系或专属道具证明是同一人。
- 角色在上下文中被明确追踪。

不能作为同一具体人物证据的情况：

- 只有相同泛称，例如 `下人`、`丫鬟`、`侍卫`、`小厮`、`黑衣人`、`乞丐`、`太医`、`百姓`。
- 只有相同职业、阶层、性别、年龄段或相似通用服装。
- 只有相同可转移称谓，且没有具体人物或时间范围消歧。
- 只有台词前缀、动作、情绪、语气或组合描述。
- 不同集或不同场只出现同类角色，没有明确连续证据。

证据不足时，默认不是同一个具体人物。

## 可见出场计数口径

人物连续性评级只统计同一具体人物的可见脸模生产出场。

可以计入人物脸模连续性的情况：

- 正常时空内的真人可见出场。
- 闪回、回忆、梦境、想象画面中需要生产可见人脸的出场。
- 画像、尸体、遗体等需要生产同一人物可见形象的出场；必须在 `source_evidence` 写明 `visual_form`。

不得计入跨集真人脸模出场的情况：

- VO、OS、旁白、画外音。
- 只在对白、心理活动或旁白中被提及。
- 牌位、坟墓、棺材、敬香、祠堂祭拜、遗物等纪念物指向。
- 泛化亲属称呼，例如父亲、母亲、阿爹、阿娘。
- 只有剧情关系存在，没有可见人物形象生产需求。

这些证据可以作为剧情、道具、场景或关系证据，但不能让人物从 `B/C` 升为 `A/S`。如果需要生产牌位、坟墓、棺材、遗物，应转入场景或道具规则。

## 硬分流规则

人物评级只允许按以下硬分流执行。不得使用其他剧情理解、主观重要性、制作便利、人物称谓印象或“看起来戏份少”等理由改写等级。

```text
if 候选未通过人物候选准入门:
    return_to_asset_extraction_or_scene_coverage
elif 不能明确认定为同一个具体人物:
    priority_level = C
elif 同一个具体人物存在跨集可见脸模生产出场:
    if 剧情重要性高 or 出现频次高 or 主线长期依赖高:
        priority_level = S
    else:
        priority_level = A
elif 同一个具体人物只在单集出现:
    if 该人物全部可见戏份能被明确归入同一个最长 15 秒的单片段:
        priority_level = C
    else:
        priority_level = B
else:
    priority_level = C
```

一句话硬规则：明确同一具体人物跨集可见出现必须评 `S/A`，再按剧情重要性和出现频次高低分 `S` 或 `A`；明确同一具体人物只在单集出现时，只有全部可见戏份能被证明处于同一个最长 15 秒单片段内才评 `C`，其余一律评 `B`；不能确认同一具体人物才评 `C`。

本规则是人物评级唯一分流入口，不得新增任何分流例外。

## 等级定义

| 等级 | 定义 | 连续性范围 | 评级含义 |
|---|---|---|---|
| `S` | 明确同一具体人物跨集可见脸模生产出场，且剧情重要性高、出现频次高或主线长期依赖高。 | 全项目固定脸 | 后续必须建立项目级人脸锚点。 |
| `A` | 明确同一具体人物跨集可见脸模生产出场，但剧情重要性、出现频次或主线依赖低于 `S`。 | 全项目固定脸 | 后续必须建立项目级人脸锚点。 |
| `B` | 明确同一具体人物只在单集出现，且不能证明全部可见戏份处于同一个最长 15 秒单片段内。 | 单集内固定脸 | 后续保持本集内脸模一致。 |
| `C` | 不能确认同一具体人物；或明确同一具体人物只在单集出现且全部可见戏份能被证明处于同一个最长 15 秒单片段内。 | 不固定脸 | 通常不需要脸模连续。 |

## 评级输出字段

`asset-rating-result` 中的人物评级项只使用以下字段：

| 字段 | 是否必填 | 允许值/规则 | 含义 |
|---|---:|---|---|
| `base_asset_id` | 是 | 稳定人物资产 ID | 被评级的人物资产 |
| `asset_type` | 是 | `character` | 资产类型 |
| `asset_name` | 是 | 规范化人物名 | 后续注册和锚定引用名 |
| `priority_level` | 是 | `S/A/B/C` | 脸模连续性等级 |
| `same_asset_evidence` | 是 | `strong/medium/weak/conflict` | 判断是否同一人的证据强度 |
| `episode_span` | 是 | 集数列表或单集 | 人物确认出现范围 |
| `scene_ids` | 是 | 场次 ID 列表 | 评级依据场次 |
| `rating_evidence_ids` | 是 | `character-evidence-ledger.evidence_id` 列表 | 支持本次评级的结构化证据回指 |
| `rating_reason` | 是 | 文本 | 为什么评为该等级 |
| `source_evidence` | 是 | 剧本证据 | 支持评级的证据 |
| `rating_uncertainty_note` | 条件必填 | 文本或空 | 同一人物证据不足、冲突或降级原因 |

`rating_evidence_ids` 不得为空；不得只写自然语言 `source_evidence` 而缺少 evidence ledger 回指。若评级所需证据尚未进入 `character-evidence-ledger`，必须阻断回 `character-evidence-ledger.md` 或 `asset-extraction.md`。

## 边界

- 不拆分人物状态。
- 不判断 `task_type`。
- 不设置 `anchor_task_id`。
- 不设置 `review_status`。
- 不生成提示词。
- 不生成分集映射。
- 不重新扫 PDF。
- 不重新切分集场。
- 不静默补写候选；候选或证据账本漏项必须阻断回上游。

只使用以上列举过的字段和职责。
