# 场景证据账本规则

## 规则目的

本文件只用于约束 `scene-evidence-ledger` 的生成方式。

本规则解决的唯一问题是：

> 在场景候选准入和 S/A/B/C 评级之前，如何无损记录全剧所有空间、地点和场景连续性证据？

本规则不替代 `资产抽取边界规则.md` 或 `场景评级规则.md`。它只生成评级前证据账本，不决定是否生产、不评级、不注册、不生成任务。

## 输入依据

- `scene-coverage-table`
- 原剧本 `source_evidence` / `source_locator`

不得重新切场。允许回看原文定位核验地点表述、空间层级、同一空间线索和泛化空间证据。

## 输出结果

输出结果固定命名为 `scene-evidence-ledger`。

## 必填字段

| 字段 | 规则 |
|---|---|
| `evidence_id` | 稳定证据 ID。 |
| `asset_type` | 固定为 `scene`。 |
| `raw_location_text` | 原文地点或空间表述。 |
| `normalized_scene_subject` | 便于候选承接的空间主体；证据不足时不要过度合并。 |
| `space_scope` | `specific_space`、`parent_location`、`generic_space`、`transition_space`、`mentioned_only`。 |
| `episode_id` | 来源集号。 |
| `scene_ids` | 来源场次 ID。 |
| `source_locator` | 页码、行号、文本块或可回溯定位。 |
| `source_evidence` | 支持该条证据的原文。 |
| `same_space_evidence_note` | 同一空间证据、归属者、房间功能、独特布置、上下场承接或证据不足说明。 |
| `visual_design_clues` | 可见空间设计线索，如牌匾、布局、材质、天气、仪式布置、破损、火灾等。 |
| `ledger_uncertainty_note` | 空间层级、泛化地点、同一空间证据不足或冲突时必填。 |

## 记录范围

- 明确地点、房间、院落、宫殿、街巷、牢房、审堂等空间。
- 泛化地点和过渡空间，例如门口、走廊、街上、屋内。
- 只在对白中提及但可能影响场景判断的地点。
- 与状态相关的空间线索，例如火灾、夜晚、封闭、宴席、祭祀、审判布置。

## 交接要求

- `asset-extraction.md` 必须从本账本读取场景证据并生成场景候选或可见排除/不确定记录。
- `asset-rating.md` 主要读取本账本进行场景评级，并可回看 `source_locator` 对应原文核验同一空间证据。
- 评级层发现本账本漏掉关键可见空间事实时，不得自行静默补齐；必须输出阻断并指回 `scene-evidence-ledger.md` 或 `scene-coverage.md`。

## 边界

- 不评级 S/A/B/C。
- 不拆分场景状态。
- 不判断 `task_type`。
- 不注册资产。
- 不生成任务、提示词或分集映射。
