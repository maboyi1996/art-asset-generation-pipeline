# 人物证据账本规则

## 规则目的

本文件只用于约束 `character-evidence-ledger` 的生成方式。

本规则解决的唯一问题是：

> 在人物候选准入和 S/A/B/C 评级之前，如何无损记录全剧所有人物相关证据？

本规则不替代 `资产抽取边界规则.md`、`人物评级规则.md` 或 `人物候选去向闭环规则.md`。它只生成评级前证据账本，不决定候选是否生产、不评级、不注册、不生成任务。

## 输入依据

- `scene-coverage-table`
- 原剧本 `source_evidence` / `source_locator`

不得绕过 `script-intake-result` 重新切场。允许回看原文定位来核验人物主体、可见性、组合关系和污染标签，但不得在本层重写其他层判断。

## 输出结果

输出结果固定命名为 `character-evidence-ledger`。

`character-evidence-ledger` 是人物评级主输入之一，也是 `asset-extraction.md` 的人物候选来源。它不是人物候选去向台账，不是生产任务表，不计入最终任务数量。

## 必填字段

| 字段 | 规则 |
|---|---|
| `evidence_id` | 稳定证据 ID。 |
| `asset_type` | 固定为 `character`。 |
| `raw_label` | 原文人物称呼、台词前缀或覆盖记录标签。污染文本只能留在本字段或原文证据。 |
| `normalized_subject` | 能可靠识别主体时填写规范主体；不能识别时留空并写明原因。 |
| `presence_type` | 继承或核验为 `physical_visible`、`memory_visible`、`dream_visible`、`portrait_visible`、`corpse_visible`、`voice_only`、`mentioned_only`、`memorial_reference`、`rumor_mentioned` 等。 |
| `episode_id` | 来源集号。 |
| `scene_ids` | 来源场次 ID。 |
| `source_locator` | 页码、行号、文本块或可回溯定位。 |
| `source_evidence` | 支持该条证据的原文。 |
| `visibility_for_rating` | `visible_face_evidence`、`non_visible_evidence`、`evidence_only`。 |
| `label_disposition` | `clean_subject`、`normalized_from_pollution`、`split_from_combination`、`generic_label`、`transferable_title`、`unresolved_label`。 |
| `same_person_evidence_note` | 同一人物证据、别名线索、稳定关系、连续事件链或证据不足说明。 |
| `ledger_uncertainty_note` | 证据冲突、主体不明、可见性不明或需要回上游复核时必填。 |

## 记录范围

- 正常可见人物、闪回/梦境/想象画面中的可见人物。
- 画像、尸体、遗体等需要人物形象生产的可见媒介。
- VO、OS、画外、对白提及、纪念物指代等非真人可见证据；这些必须记录，但不得作为跨集真人脸模升级证据。
- 泛称人物、群体人物、背景功能人物。
- 台词前缀污染、动作/情绪/语气/声音修饰、组合短语、未消歧可转移称谓。

## 清洗与保留规则

- `皇帝点头`、`棠宁喃喃`、`皇后拍桌` 等写入 `raw_label`，`normalized_subject` 写真实主体，`label_disposition` 写 `normalized_from_pollution`。
- `二夫人三夫人一起` 等组合标签能拆分时写多条证据或一条拆分说明，`label_disposition` 写 `split_from_combination`，不得形成组合人物资产。
- `四夫人`、`夫人`、`主子` 等可转移称谓必须记录上下文和时间范围；无法绑定具体人物时写 `transferable_title` 或 `unresolved_label`。
- 泛称人物只记录为泛称证据，不因跨场或跨集重复自动确认为同一个具体人。

## 交接要求

- `asset-extraction.md` 必须从本账本读取人物证据并生成 `asset-candidate-list` 或污染清洗/阻断记录。
- `asset-rating.md` 主要读取本账本进行人物硬分流，并可回看 `source_locator` 对应原文核验。
- 评级层发现本账本漏掉可见人物事实时，不得自行静默补齐；必须输出阻断并指回 `character-evidence-ledger.md` 或 `scene-coverage.md`。

## 边界

- 不评级 S/A/B/C。
- 不判断 `task_type`。
- 不设置 `review_status`。
- 不生成候选最终去向。
- 不注册资产。
- 不生成提示词或分集映射。
