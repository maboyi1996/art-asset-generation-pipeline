# 提示词语言守卫

## 目的

保证人物、场景、道具的生产提示词可以作为中文 core prompt 进入中台，并等待中台拼接画风前缀。

本守卫只检查语言和表达边界，不重写提示词、不补写风格、不选择画法。

## 语言规则

- 人物、场景、道具最终 `prompt` 必须使用中文。
- 人物状态 `edit_instruction` 必须使用中文。
- `negative_prompt` 必须使用中文负向约束。
- 不得残留旧版英文主模板句式。
- 可保留必要的数字、比例、ID、颜色值和极少量技术字段名，例如 `9:16`、`16:9`、`#808080`、`image_text_edit`。
- 中文剧本证据可被转写成可见画面描述，但不得把整段原文粗暴粘进提示词。

## 结构规则

提示词应包含：

- 资产身份。
- 状态名称对应的可见差异。
- 剧情功能或空间/物件身份。
- 关键外貌、材质、光线、构图或版式。
- 负向约束。

## 禁止

- 只写一句摘要式提示词。
- 用剧情事件替代可见画面。
- 用“很美”“高级”“氛围感”代替具体视觉信息。
- 为人物主图加入道具、场景、动作戏或其他角色。
- 写入 `【A. 题材世界】`、`【B. 画法】` 或 `{{include:...}}`。
- 写入由中台负责拼接的画风前缀。
