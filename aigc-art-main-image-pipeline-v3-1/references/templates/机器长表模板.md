# 机器生产源表模板

## 文件名

`production-source/main-image-production-table.md`

## 用途

本文件是 JSON 和 DOCX 的唯一机器生产源表。V3.1 中它是瘦生产源表，不再承载完整中间审计包。

完整证据账本、候选去向、污染清洗、状态线索去向、任务草稿和生产审计明细必须保存在 `audit/` 独立文件中，并在 `artifact-manifest.json` 登记。

## 必须包含

1. 项目概览。
2. 全局资产注册表。
3. 状态索引表。
4. 角色身份锚点表。
5. 全局生产任务表。
6. 分集使用映射表。
7. 生产审计摘要。

不得包含完整的：

- 人物评级前证据账本。
- 场景评级前证据账本。
- 道具评级前证据账本。
- 别名归并表全文。
- 状态线索去向表全文。
- 人物候选去向审计表全文。
- 泛称/群体模板覆盖表全文。
- 污染标签清洗记录表全文。

如需人工或维护者排查这些内容，读取 `audit/` 对应文件。

## 全局生产任务表字段

| 字段 |
|---|
| `task_id` |
| `base_asset_id` |
| `state_asset_id` |

| `state_name` |

| `task_type` |
| `anchor_task_id` |
| `review_status` |
| `review_reason` |
| `episode_id` |
| `source_evidence` |
| `prompt` |
| `edit_instruction` |
| `negative_prompt` |
| `review_focus` |

全局生产任务表字段只使用上表列举字段；新增字段必须先更新 `任务类型字段审核规则.md`、`validate_production_source.py`、`generate_prompt_manifest.py` 和 `generate_review_docx.py`。

`base_asset_id` 必须回指全局资产注册表。`state_asset_id` 仅状态任务填写；基础任务为空。JSON 生成层只能通过这两个稳定 ID 字段投影 `prompt-manifest.json`，不得用临时拼接 ID 或仅靠资产名猜测归属。

所有来自 `audit/global-task-draft.md` 且进入生产的任务行必须进入本表；人物候选去向审计表、污染标签清洗记录表、角色身份锚点表和分集使用映射表都不得替代本表。

`text_to_image` 任务必须填写 `prompt` 和 `negative_prompt`。`image_text_edit` 任务必须填写 `edit_instruction`、`negative_prompt` 和有效 `anchor_task_id`。

V3.1 字段语义：

- `prompt` 是由 `render_core_prompts.py` 渲染的中文 core prompt，不是完整画风提示词。
- `edit_instruction` 是由 `render_core_prompts.py` 渲染的中文状态编辑 core instruction。
- `negative_prompt` 是脚本渲染或生产源固定输出的中文负向约束。

以上字段不得包含 `【A. 题材世界】`、`【B. 画法】`、`{{include:...}}`、风格前缀或旧英文写实模板残留。
