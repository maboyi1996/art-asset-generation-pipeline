# 道具主图提示词模板

## 用途

供 `scripts/render_core_prompts.py` 根据 `production-source/core-prompt-slots.md` 渲染道具资产中文 core prompt。V3.1 中 `prompt-generation.md` 只填写槽位，不直接手写整段 prompt。

本模板不包含题材世界段、画法段或风格前缀。中台会在本 core prompt 前拼接运营选择的风格前缀。

## 核心原则

本模板原始设计有 A「题材世界」和 B「道具画法」两层；V3.1 改为由中台后续拼接，所以模板正文只生成 C 之后的 core prompt。D/E 只写道具身份、结构、基础材质、颜色、状态和使用痕迹，不写题材词、画法词、渲染词或风格前缀。C 段版式/背景规则和末尾负向约束是生产防错位置；D/E 段不得写否定、排除、纠错或对照句式。缺失元素直接省略，不声明其不存在。负向约束只负责生产防错，不承担题材或画法控制。

## 输入字段

- `object_identity`
- `prop_name`
- `ownership_usage`
- `story_function`
- `one_sentence_positioning`
- `required_elements`
- `optional_details`
- `overall_condition`

## 主图 core prompt 模板

```text
图像类型：
道具资产参考图。

画幅比例：
横版 16:9。

版式规则：
左右两块并列展示，每块约 8:9。左侧为道具正面图，右侧为道具背面图。

背景规则：
#808080 色号代表的干净中性灰底图。

道具名称：
[填写道具名称]

准确道具身份：
[具体道具实体类型 / 核心结构 / 归属对象或势力 / 实际用途 / 一眼识别特征]

归属与用途：
[所属世界 / 所属角色或势力 / 阶层属性 / 技术等级 / 实际用途]

剧情功能：
[线索 / 武器 / 交易物 / 信物 / 身份凭证 / 情绪锚点 / 秘密载体]

一句话定位：
[精准道具定位 + 归属对象或势力 + 实际用途 + 核心识别特征]

必须出现：
- [核心部件 / 识别细节 + 材质 + 颜色 + 新旧状态 / 使用痕迹]
- [核心部件 / 识别细节 + 材质 + 颜色 + 新旧状态 / 使用痕迹]

可选细节：
- [辅助结构 / 功能部件 / 装饰细节 + 材质 + 颜色 + 新旧状态 / 使用痕迹]
- [辅助结构 / 功能部件 / 装饰细节 + 材质 + 颜色 + 新旧状态 / 使用痕迹]

整体状态：
[全新 / 使用过但保养好 / 陈旧 / 破损 / 潮湿 / 沾血 / 烧灼 / 油污 / 刮花 / 氧化]

负向约束：
单个道具主体，不要环境，不要人物，不要手，不要变成剧情截图或手持使用状态；不要错误道具类型；不要随机增加无关物件；不要让核心结构和材质状态消失；不要商品广告图；不要可读文字、真实品牌标识或水印。
```

## 约束

- 最终提示词必须是中文 core prompt。
- 道具图默认独立灰底单物。
- 不自动加入人物、手、环境或品牌文字。
- D/E 段只写正向可见事实，不写否定、排除、纠错或对照句式。
- 缺失元素直接省略，不写不存在元素的否定声明。
- 防人物、防手、防环境、防错道具、防商品广告等生产防错只写在负向约束或 `negative_prompt`。
- 一句话定位必须是正向定位句，不写纠错对照句。
- 不写题材世界段、画法段、`{{include:...}}` 或风格库前缀。
