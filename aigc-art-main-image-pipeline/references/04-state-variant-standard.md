# 主图资产状态判定标准

本标准用于决定角色、场景、道具是否需要拆成多个“主图状态资产”。核心目标是避免漏掉真正会影响美术一致性的外观变化，同时禁止把剧情节点、情绪、道具或场景误当成人物状态。

## 0. 总原则

状态资产必须满足三个条件：

1. 可见：状态变化必须能在静态主图里被直接看见。
2. 稳定：状态变化不是一瞬间的动作或表情，而是足以指导后续画面一致性的外观条件。
3. 必要：如果不单独出主图，后续生成会把不同外观混在一起，影响角色、空间或道具识别。

不满足以上三条的内容，只能写进剧情证据、审核重点或场景/道具说明，不能拆状态。

## 1. 人物状态判定

人物状态只按“人物本体外观”拆分。人物状态不包含道具、场景、站位、动作、剧情节点或单纯情绪。

### 1.1 必须拆分的人物状态

以下变化会明显改变角色外观，通常必须拆状态：

| 变化类型 | 必须拆分条件 | 示例状态名 |
|---|---|---|
| 年龄变化 | 少年/青年/中年/老年，或剧情中明显衰老、返老、回忆年轻版 | 外观（青年）状态、外观（白发衰老）状态 |
| 发型发色 | 黑发变黄发、长发变短发、剃光头、发髻/披发等轮廓明显变化 | 外观（黄发）状态、外观（光头）状态 |
| 胡须/面部毛发 | 刮净、络腮胡、胡茬、长须等影响脸部识别 | 外观（胡茬疲惫）状态 |
| 妆容 | 素颜、浓妆、舞台妆、婚礼妆、毁妆、病态妆 | 外观（素颜）状态、外观（毁妆）状态 |
| 服装类别 | 日常服、工作服、制服、正装、白大褂、防护服、睡衣、婚服、丧服、伪装服等 | 外观（研发白大褂）状态、外观（朴素正装）状态 |
| 身体状态 | 生病、受伤、流血、脏污、湿发、醉态、疲惫、营养不良、怀孕等可见身体变化 | 外观（乱发浮肿醉态）状态、外观（受伤虚弱）状态 |
| 社会身份外观 | 贫穷落魄、暴富精致、职业身份上身、阶层外观明显变化 | 外观（白发破衣终局）状态、外观（精致资本老板）状态 |
| 特殊身份伪装 | 女扮男装、改名换装、遮脸口罩帽子、制服伪装等 | 外观（帽子口罩夜行）状态 |

### 1.2 可以合并的人物状态

以下变化通常不单独拆状态，除非剧本明确要求作为视觉重点：

| 内容 | 默认处理 |
|---|---|
| 同一套衣服只是脏一点、皱一点 | 合并到同一状态，在 prompt 中写“slightly worn / wrinkled” |
| 同一状态下不同情绪，如愤怒、得意、难过 | 不拆状态，写进表情控制或审核重点 |
| 同一人物在不同地点出现但外观不变 | 不拆状态，地点由场景资产负责 |
| 同一人物拿不同道具但外观不变 | 不拆状态，道具另建道具资产 |
| 同一服装只换颜色但轮廓、身份功能不变 | 原则上合并，除非颜色是剧情识别点 |
| 轻微姿势差异，如坐、站、回头、跪地 | 不拆状态，动作不属于人物外观主图 |

### 1.3 禁止误拆的人物状态

以下绝不应作为人物状态名：

| 错误状态 | 错误原因 | 正确归属 |
|---|---|---|
| 坐在120万现金中间 | 这是场景和道具证据，不是人物外观 | 现金道具、客厅场景 |
| 拿着合同 | 道具行为，不改变人物本体外观 | 合同道具 |
| 站在酒厂里 | 地点信息，不改变人物本体外观 | 酒厂场景 |
| 和某人对峙 | 剧情关系，不是外观状态 | 剧情证据 / 审核重点 |
| 愤怒状态 / 开心状态 | 单纯情绪，通常不是稳定外观 | 表情描述 |
| 被羞辱状态 | 抽象剧情处境，不是可见造型 | 剧情证据 |
| 分钱状态 / 签约状态 / 举报状态 | 剧情事件，不是人物外表 | 场景或道具任务 |

### 1.4 人物主图 prompt 要求

人物主图必须满足：

- 英文生图 prompt。
- 单人。
- 纯色或中性干净背景。
- 上半身、半身或肖像，脸和服装状态清晰。
- 不放场景。
- 不放道具。
- 不放其他人物。
- 不写中文剧情证据。
- 不把“证据句”塞进 prompt。

人物 prompt 可以描述：

- 年龄感。
- 脸型、皮肤、眼神、疲惫/病态等可见状态。
- 发型、发色、胡须、妆容。
- 衣服类别、材质、磨损程度。
- 身体姿态气质，但不能变成动作场景。

人物 prompt 不应描述：

- 现金、合同、酒坛、检测仪、手机、三轮车等道具。
- 客厅、酒厂、论坛、田地、桥洞等场景。
- “E01-S001 里发生了什么”这种中文剧情证据。
- 和谁发生关系或冲突。

### 1.5 人物状态命名

格式：

`角色名--外观（XX）状态`

命名规则：

- `XX` 必须是外表可见内容。
- 优先使用短词，不写剧情句。
- 不写地点、道具、事件。

好命名：

- `陈有财--外观（中年乡镇老板）状态`
- `陈有财--外观（乱发浮肿醉态）状态`
- `陈有财--外观（白发破衣终局）状态`
- `陈北--外观（研发白大褂）状态`
- `阿强--外观（帽子口罩夜行）状态`

坏命名：

- `陈有财--外观（坐在120万现金中间）状态`
- `陈北--外观（拒绝五千万收购）状态`
- `陈小海--外观（被警察带走）状态`
- `钱万里--外观（论坛被打脸）状态`

## 2. 场景状态判定

场景状态只按“空间本体的可见状态”拆分。场景状态不因人物情绪、对白内容或镜头角度变化而拆分。

### 2.1 必须拆分的场景状态

| 变化类型 | 必须拆分条件 | 示例状态名 |
|---|---|---|
| 时间/光线 | 日景、夜景、停电、晨光、黄昏会明显改变空间识别 | 初始（日间生产）状态、初始（夜间停电）状态 |
| 破坏/打砸 | 门窗、家具、墙面、设备等被打砸后空间状态明显改变 | 初始（打砸后）状态 |
| 查封/封锁 | 封条、警戒线、停业状态改变空间识别 | 初始（查封后）状态 |
| 活动布置 | 开业、论坛、宴会、发布会、合作社成立等布置改变空间功能 | 初始（开业剪彩）状态 |
| 工地/建成 | 同一地点从空地、工地到建成厂区 | 初始（工地开工）状态、初始（建成投产）状态 |
| 拥挤/空置 | 人群、货物、设备让空间功能发生明显变化 | 初始（满负荷生产）状态 |
| 污染/事故 | 火灾后、水淹后、污染后、酒液流出等改变环境质感 | 初始（污染事故后）状态 |

### 2.2 可以合并的场景状态

| 内容 | 默认处理 |
|---|---|
| 同一房间里换一段对白 | 不拆 |
| 人物站位或坐位变化 | 不拆 |
| 镜头角度变化 | 不拆 |
| 临时放了一个小道具，不改变空间识别 | 不拆，道具另建 |
| 气氛从紧张变开心，但灯光/陈设没变 | 不拆 |

### 2.3 场景主图 prompt 要求

场景主图可以包含：

- 空间类型。
- 空间层级和用途。
- 墙面、地面、天花、家具、设备、固定陈设。
- 光线和时间。
- 关键空间锚点。
- 必要时允许少量人物作为尺度参照，但不做人物设计。

场景主图不应变成：

- 人物肖像。
- 动作戏定格。
- 道具产品图。
- 只讲情绪的氛围图。

### 2.4 场景状态命名

格式：

`场景名--初始（XX）状态`

好命名：

- `陈北酒厂--初始（日间抽检）状态`
- `陈有财酒坊门口--初始（查封围堵）状态`
- `行业论坛现场--初始（公开对质）状态`
- `陈北新酒厂工地--初始（开工剪彩）状态`

坏命名：

- `陈北酒厂--初始（陈北很愤怒）状态`
- `办公室--初始（钱万里威胁陈北）状态`
- `客厅--初始（陈小海不服）状态`

## 3. 道具状态判定

道具状态只按“物体本身的可见条件”拆分。谁拿着它、它放在哪里、镜头从哪个角度拍，通常不构成状态。

### 3.1 必须拆分的道具状态

| 变化类型 | 必须拆分条件 | 示例状态名 |
|---|---|---|
| 新旧变化 | 新道具和旧损道具外观差异明显 | 初始（全新）状态、初始（旧损）状态 |
| 打开/封存 | 打开后内部结构或内容物是剧情重点 | 初始（封存）状态、初始（打开）状态 |
| 破损 | 碎裂、断裂、变形影响识别 | 初始（破碎）状态 |
| 污染 | 沾血、沾泥、霉变、酒液污染、工业酒精污染等 | 初始（受潮发霉）状态 |
| 内容物变化 | 瓶中液位、文件内容、箱内物品对剧情重要 | 初始（半瓶投毒）状态 |
| 标记变化 | 红章、指印、签名、封条、标签等视觉标记改变道具功能 | 初始（血红指印）状态 |
| 所有权外观变化 | 转手后包装、贴纸、封条、破损痕迹发生变化 | 初始（转让后）状态 |

### 3.2 可以合并的道具状态

| 内容 | 默认处理 |
|---|---|
| 同一完好物体放在桌上或地上 | 不拆 |
| 谁拿着它 | 不拆 |
| 拍摄角度不同 | 不拆 |
| 同一文件被不同人物看 | 不拆 |
| 同一瓶酒在不同场景出现但外观不变 | 不拆 |
| 道具只有象征意义，没有外观差异 | 不拆 |

### 3.3 道具主图 prompt 要求

道具主图必须满足：

- 单一主体。
- 干净白底或中性背景。
- 不出现人物手。
- 不出现原场景。
- 不出现无关物品。
- 不出现真实品牌 logo。
- 文字除非是剧本明确要求的品牌牌匾，否则尽量用“文字形状/红章形状/标签形状”避免生成乱码。

### 3.4 道具状态命名

格式：

`道具名--初始（XX）状态`

好命名：

- `发霉玉米麻袋--初始（受潮霉变）状态`
- `工业酒精瓶--初始（半瓶投毒）状态`
- `陈记酒坊转让协议--初始（血红指印）状态`
- `北哥纯粮酒陶坛--初始（温度标签）状态`

坏命名：

- `合同--初始（钱万里很嚣张）状态`
- `酒碗--初始（陈北很感动）状态`
- `三轮车--初始（陈有财遇见陈北）状态`

## 4. 资产层级边界

### 4.1 人物、场景、道具的职责

| 资产类型 | 负责内容 | 不负责内容 |
|---|---|---|
| 人物 | 外貌、年龄、发型、妆容、服装、身体状态、气质 | 道具、场景、剧情事件、其他人物 |
| 场景 | 空间结构、陈设、光线、时间、空间状态 | 人物脸部设计、单一道具产品图 |
| 道具 | 物体类型、结构、材质、状态、标记 | 人物外貌、空间布局、动作戏 |

### 4.2 冲突处理

如果一个视觉信息可以归属多个层级，按以下顺序判断：

1. 信息是否改变人物身体或服装？是，则归人物。
2. 信息是否改变空间结构或空间状态？是，则归场景。
3. 信息是否改变物体本身？是，则归道具。
4. 只是剧情关系、行为或情绪？不建状态，只写证据/审核重点。

示例：

| 源文本信息 | 正确归属 | 说明 |
|---|---|---|
| 陈有财坐在现金中间 | 场景 + 道具 | 人物外观未变 |
| 陈有财头发全白、衣服破旧 | 人物 | 外貌和服装变化明显 |
| 陈北手持酒碗演讲 | 道具 + 场景 | 人物可用“朴素正装”，酒碗另建 |
| 陈北穿白大褂研发 | 人物 | 服装类别改变 |
| 酒厂夜间被投工业酒精 | 场景 + 道具 | 夜间发酵池是场景，工业酒精瓶是道具 |
| 论坛大屏显示检测报告 | 场景 + 道具 | 论坛是场景，报告是道具 |

## 5. 证据强度标准

每个状态必须标注证据强度：

| 等级 | 定义 | 是否可直接出主图 |
|---|---|---|
| 强证据 | 剧本直接写明外观、服装、状态、空间变化或道具条件 | 可以 |
| 中证据 | 由明确场景/职业/动作强推断，且符合常识 | 可以，但审核重点标注“推断” |
| 弱证据 | 只有剧情情绪或抽象身份，没有可见描述 | 不应拆状态 |
| 冲突证据 | 不同段落描述互相矛盾 | 标记待人工确认，不直接出 prompt |

人物状态必须至少有“强证据”或“中证据”。仅凭“他很生气”“他被羞辱”“他胜利了”不能拆人物状态。

## 6. 状态拆分决策流程

逐条资产按以下流程判断：

1. 这条信息属于人物、场景还是道具？
2. 它是否可见？
3. 它是否改变该资产的本体外观？
4. 它是否持续到足以影响后续画面一致性？
5. 如果不拆，后续生图是否会混淆？
6. 是否有足够剧本证据？
7. 拆分后状态名是否只写可见外观/空间/物体条件？

只有 2-7 都通过，才拆状态。

## 7. 审计清单

输出前逐项检查：

- 人物状态名里是否出现地点？如果有，改。
- 人物状态名里是否出现道具？如果有，改。
- 人物状态名里是否出现剧情事件？如果有，改。
- 人物 prompt 是否有中文？如果有，删。
- 人物 prompt 是否出现现金、文件、酒坛、车辆、场景？如果有，移到场景/道具。
- 场景状态是否只是对白或情绪变化？如果是，合并。
- 道具状态是否只是换了持有人或摆放位置？如果是，合并。
- 每个状态是否有场次证据？没有则标记待人工确认。
- B/C 级状态是否真的需要主图？如果不需要，保留记录即可。

## 8. 本项目应用示例

### 8.1 陈有财

正确人物状态：

- `陈有财--外观（中年乡镇老板）状态`
- `陈有财--外观（乱发浮肿醉态）状态`
- `陈有财--外观（黑眼圈衰老）状态`
- `陈有财--外观（白发破衣终局）状态`

不应作为人物状态：

- `分钱状态`
- `坐在120万现金中间`
- `跪地求陈北`
- `推三轮车捡废品`

说明：

- “白发、破衣、沟壑纵横”是人物外观，归人物。
- “120万现金”是道具。
- “客厅分钱”是场景。
- “破三轮车”是道具。

### 8.2 陈北

正确人物状态：

- `陈北--外观（辛劳酿酒匠）状态`
- `陈北--外观（安全帽创业者）状态`
- `陈北--外观（研发白大褂）状态`
- `陈北--外观（朴素正装）状态`

不应作为人物状态：

- `拒绝五千万收购`
- `直播澄清`
- `合作社成立`
- `坟前祭祖`

说明：

- “安全帽”“白大褂”“正装”是服装/身份外观变化，归人物。
- “直播间”“合作社舞台”“爷爷坟前”是场景。
- “检测报告”“酒碗”是道具。

### 8.3 场景示例

正确场景状态：

- `陈北酒厂--初始（日间抽检）状态`
- `陈北酒厂夜间发酵池--初始（工业酒精投毒）状态`
- `行业论坛现场--初始（公开对质）状态`
- `陈有财酒坊门口--初始（查封围堵）状态`

不应作为场景状态：

- `陈北很愤怒`
- `钱万里尴尬`
- `小李很急`

### 8.4 道具示例

正确道具状态：

- `2万与60万现金--初始（分红对比）状态`
- `发霉玉米麻袋--初始（受潮霉变）状态`
- `工业酒精瓶--初始（半瓶投毒）状态`
- `陈记酒坊转让协议--初始（血红指印）状态`

不应作为道具状态：

- `现金--初始（陈北被羞辱）状态`
- `合同--初始（钱万里威胁）状态`
- `酒碗--初始（陈北感动）状态`

## 9. 输出字段建议

每个状态资产建议保留以下字段：

- `asset_id`
- `base_asset_id`
- `asset_type`
- `display_name`
- `state_name`
- `state_basis_type`: 外观 / 空间 / 物体条件
- `state_evidence_scene_ids`
- `state_evidence_summary`
- `evidence_strength`: strong / medium / weak / conflict
- `split_reason`
- `merge_or_exclusion_reason`
- `prompt_scope`
- `negative_prompt_boundary`
- `review_focus`

人物 `prompt_scope` 固定为：

`single-character portrait, plain solid background, no props, no environment`

场景 `prompt_scope` 固定为：

`main scene reference, readable spatial layout`

道具 `prompt_scope` 固定为：

`isolated prop reference, clean background`
