# 输出质量守卫

## 目的

检查最终交付是否满足 V2.53 输出契约。

## 必查项

- 是否输出 `main-image-production-table.md`。
- 是否输出 `main-image-review-table.md`。
- 机器长表是否包含人物候选去向审计表、全局生产任务表和分集使用映射。
- 人审短表是否只包含项目概览、角色身份锚点短表、分集使用映射。
- 人审短表是否没有放入完整人物候选去向台账。
- `character-disposition-ledger` 是否覆盖所有 `asset_type: character` 候选。
- 每个人物候选是否都有合法 `final_disposition`。
- 是否存在 B/C 人物没有生产、状态、模板、别名、有效分组、排除或非视觉去向。
- 是否存在 B/C 可生产人物只出现在人物候选去向审计表、没有进入全局生产任务表。
- B 级可视人物是否有 `text_to_image` 任务和 `episode_id`。
- B 级人物数量为 0 时，是否有明确审计说明证明没有同一具体人物、单集内、非单片段的候选；否则不得直接通过。
- C 级人物是否有正向 C 级证据：单片段、单镜、背景、群体、泛称、只提及、无法确认同一具体人物，或无需单人脸模连续。
- C 级可视单人或模板人物保留为资产时，是否有 `text_to_image` 任务或合法可见排除记录。
- `production_task`、`state_task`、`template_task` 去向是否都能找到对应 `task_id`。
- `manual_review_required` 是否只作为任务行 `review_status` 使用，而不是替代应生产资产的 `task_id`。
- `grouped_into` 是否指向真实模板任务、集合任务或可见排除记录。
- B/C 人物数量异常为 0 时，是否有明确审计说明。
- 是否把 `character-anchor-result` 误当作完整人物清单。
- 人物候选去向审计字段是否没有污染全局生产任务表字段。
- 每个 `task_id` 是否稳定且可被分集映射引用。
- `task_type` 是否只使用 `text_to_image` 和 `image_text_edit`。
- `review_status` 是否只使用 `ready`、`manual_review_required`、`excluded`。
- 状态是否继承 base asset 等级。
- 人物状态图是否正确引用 `anchor_task_id`。
- 每个 `text_to_image` 任务是否填写 `prompt` 和 `negative_prompt`。
- 每个 `image_text_edit` 任务是否填写 `edit_instruction`、`negative_prompt` 和有效 `anchor_task_id`。
- 提示词层是否没有重新评级。
- 测试报告是否包含人物候选 S/A/B/C 数量、人物生产任务 S/A/B/C 数量、各等级 `text_to_image` 数量、各等级 `image_text_edit` 数量。
- 测试报告是否包含场景和道具的 S/A/B/C 任务数量。
- 测试报告是否包含 B=0 异常说明、C 级正向证明抽查和 B/C 边界样本。

## 失败处理

发现结构缺失、人物候选未闭环、应生产资产缺任务、断链、字段非法、提示词缺失或人审短表影响机器长表时，返回对应拥有层修复，不在最终质检层重写结果。
