# 状态拆分规则

## 规则目的

本文件用于判断一个 base asset 是否需要拆成可见状态资产。

本文件只定义状态边界与状态继承，不定义资产等级、任务字段、输出表结构或提示词模板。

## 核心规则

状态资产是同一个 base asset 的可见差异生产目标。

拆状态必须同时满足：

1. 差异能在静态主图中看见。
2. 差异改变的是资产自身的外观、空间条件或物体条件。
3. 差异稳定到足以指导生产，不是瞬时动作、表情或镜头角度。
4. 不拆会导致后续生成混淆不同版本。
5. 有剧本证据，或明确标记证据不足。

## 状态继承

一个 base asset 只能有一个 `priority_level`。

所有状态资产继承 base asset 的 `priority_level`。

不得对状态重新评级。

不得新增 `state_importance`、`key_state`、`minor_state` 等字段。只要状态需要生成，就已经具备生产意义。

## 人物状态边界

人物状态只包含人物本体可见外观变化：

- 年龄外观。
- 脸部或胡须等修饰变化。
- 发型或发色。
- 妆容。
- 服装类别、轮廓、制服、伪装、礼服、工服。
- 伤病、疲惫、脏污、血迹、湿发、怀孕、可见身体状态。
- 阶层/身份呈现导致的可见造型变化。
- 闪回/过去时期的可见外观，如童年、青年、老年、过去工装、过去贫穷或富贵造型。

人物状态不包含：

- 拿着某个道具。
- 站在某个地点。
- 争吵、签字、跪地、举报、复仇、受辱、胜利、失败。
- 与另一个人物的关系。
- 没有稳定外观变化的瞬时情绪。

## 场景状态边界

场景状态只包含空间条件的可见变化：

- 昼夜或光照条件显著改变空间身份。
- 停电、应急照明、封闭/封锁状态。
- 活动布置、宴席布置、审堂布置、仪式布置。
- 损毁、打砸、火灾、水淹、污染、事故前后。
- 建设/完工状态。
- 拥挤/空置、满载/清空等改变空间功能的状态。
- 闪回/过去时期的空间条件。

场景状态不包含：

- 对白内容。
- 人物情绪。
- 镜头角度。
- 临时站位。
- 不改变空间的小道具摆放。

## 道具状态边界

道具状态只包含物体条件的可见变化：

- 新旧、完整/破损、干净/脏污、潮湿/烧毁/发霉/沾血。
- 封存/打开，且内部结构或内容重要。
- 标签、盖章、签名、指印、封条、液位、可见内容变化。
- 被改造、隐藏、武器化、污染，或转手后带有可见标记。

道具状态不包含：

- 谁拿着它。
- 它被放在哪里。
- 镜头角度。
- 只有象征意义。
- 人物对它的情绪反应。

## 状态命名

使用短、可见、具体的状态名：

- 人物：`角色名--外观（XX）状态`
- 场景：`场景名--初始（XX）状态`
- 道具：`道具名--初始（XX）状态`

`XX` 必须是可见、具体、对生产有用的描述。

## 状态复用

同一 base asset 的同一可见状态可以复用同一个状态任务。

状态复用必须同时满足：

1. `base_asset_id` 相同。
2. `state_name` 指向同一种可见状态。
3. `visual_difference` 没有新增生产差异。
4. 状态证据不冲突。
5. 复用范围不超过该 base asset 的 `priority_level` 连续性范围。

只要外观、空间条件或物体条件发生新的可见生产差异，就必须创建新的状态资产。

人物状态是否需要锚定图、是否使用 `image_text_edit`、是否引用 `anchor_task_id`，由 `角色锚定规则.md` 和 `任务类型字段审核规则.md` 处理。本文件只判断状态是否应该拆分和复用。

## 输出要求

每个状态候选至少携带：

- `base_asset_id`
- `state_asset_id`
- `state_name`
- `asset_type`
- `state_basis_type`: `appearance`、`space_condition`、`object_condition`
- `state_trigger_scene_ids`
- `episode_list`
- `scene_list`
- `state_evidence`
- `evidence_strength`: `strong/medium/weak/conflict`
- `visual_difference`
- `split_reason`
- `state_uncertainty_note`，如需要

## 边界

- 不定义资产等级。
- 不判断 `task_type`。
- 不设置 `anchor_task_id`。
- 不设置 `review_status`。
- 不生成提示词。
- 不生成分集映射。

只使用以上列举过的字段和职责。
