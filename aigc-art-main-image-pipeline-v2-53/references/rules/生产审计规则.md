# 生产审计规则

## 规则目的

本文件只用于 `production-audit.md` 的生产前一致性审计。

生产审计的核心问题是：

> 前序结构化结果是否完整、一致、可被提示词生成层安全读取？

本文件只定义审计对象、审计问题类型、严重程度和 `production-audit-result` 输出字段。

本文件不重新抽取资产、不重新拆状态、不重新评级、不重写提示词、不直接修复注册表或分集映射。

## 审计对象

| 审计对象 | 检查内容 |
|---|---|
| `scene-coverage-table` | 场次事实是否被候选资产、映射或可见排除记录覆盖 |
| `asset-candidate-list` | 候选资产是否都有注册、合并、排除或审核去向 |
| `state-variant-result` | 状态是否挂到正确 base asset，是否存在状态孤儿 |
| `asset-rating-result` | 每个 base asset 是否有唯一 `priority_level`，状态是否继承等级 |
| `global-asset-registry` | 资产 ID、状态 ID、别名合并是否稳定 |
| `global-task-draft` | 应生产资产是否都有任务行，`task_id`、`task_type`、`review_status`、`anchor_task_id` 是否完整合法 |
| `character-disposition-ledger` | 每个人物候选是否有最终去向，B/C 是否未被静默丢失，去向是否引用真实任务或可见排除记录 |
| `episode-usage-map-draft` | 分集映射是否能引用到存在的 `task_id` |
| `character-anchor-result` | S/A 人物是否有锚点任务，状态图是否依赖锚点；不得当作完整人物清单 |

## 问题类型

| 问题类型 | 审计动作 |
|---|---|
| 缺必要字段 | 输出审计问题，指向负责字段的上游层级 |
| `task_id` 断链 | 输出阻断问题，要求回到全局任务草稿或分集映射层处理 |
| S/A 人物缺锚点 | 输出阻断问题，要求回到角色锚定层处理 |
| 状态孤儿 | 输出阻断问题，要求回到状态拆分或资产注册层处理 |
| 候选资产无去向 | 输出覆盖问题，要求回到资产注册层处理 |
| 应生产资产缺任务 | 输出阻断问题；要求回到资产注册层按 `全局资产任务生成规则.md` 补齐 `global-task-draft` |
| 人物候选无最终去向 | 输出阻断问题，要求回到资产注册层补齐 `character-disposition-ledger` |
| B/C 可生产人物缺 `text_to_image` 任务 | 输出阻断问题；B/C 可生产人物必须有生产任务，不能只存在于人物候选去向台账 |
| B/C 人物无闭环 | 输出阻断问题；B/C 人物必须有真实生产任务、状态任务、模板任务、别名、有效分组、排除或非视觉去向 |
| B 级人物数量异常为 0 | 输出警告或阻断问题；必须要求回到评级结果核查是否把单集多片段风险人物误判为 C，不能用空 B 结果直接通过 |
| 锚点结果误作完整人物清单 | 输出阻断问题，要求回到输出契约或资产注册层，不能用 `character-anchor-result` 代替人物候选台账 |
| 人物去向引用断链 | 输出阻断问题；`task_id`、`alias_to`、`grouped_into` 必须能引用到存在目标 |
| 人工审核状态替代任务 | 输出阻断问题；`manual_review_required` 只能作为任务行 `review_status`，不能替代应生产资产的 `task_id` |
| 空分组替代任务 | 输出阻断问题；`grouped_into` 必须指向真实模板任务、集合任务或可见排除记录 |
| 泛称人物粗暴合并 | 输出警告或阻断问题；缺少同一人物证据时要求回到资产注册层拆分、分组、模板化或排除 |
| 分集映射缺引用 | 输出映射问题，要求回到分集映射生成位置处理 |
| 审核状态与任务字段冲突 | 输出字段问题，要求回到任务类型字段审核规则处理 |

生产审计只能给出 `required_action`，不得在本层直接补字段、补注册、补人物任务、改评级、改状态或重写提示词。

## 严重程度

| `severity` | 含义 | 是否可进入提示词生成 |
|---|---|---:|
| `blocking` | 会导致任务断链、锚点缺失、字段无法读取或资产去向缺失。 | 否 |
| `warning` | 可以生成，但需要人工确认或后续复核。 | 是 |
| `info` | 不影响生产，只用于说明。 | 是 |

## 输出要求

`production-audit-result` 至少包含：

- `audit_status`
- `issue_id`
- `issue_type`
- `affected_layer`
- `affected_id`
- `severity`
- `required_action`
- `suggested_review_status_update`
- `review_reason`

审计结果必须被 `prompt-generation.md`、`output-contracts.md` 和 `final-quality.md` 读取。

## 任务完整性审计

生产审计必须用 `asset-rating-result`、`global-asset-registry`、`global-task-draft` 和 `character-disposition-ledger` 交叉检查：

- S/A/B/C 已注册且应生产的角色、场景、道具资产是否都有 `global-task-draft` 行。
- B 级人物基础形象是否都有 `text_to_image` 任务和 `episode_id`。
- B 级人物数量为 0 时，是否有评级层或审计摘要说明：候选人物中没有明确同一人、单集内、非单片段的人物；如无说明，必须标记为至少 `warning`，对长剧或人物候选较多项目可标记为 `blocking`。
- C 级可视单人或模板人物如果保留为资产，是否有 `text_to_image` 任务或合法可见排除记录。
- C 级人物是否有正向 C 级证据：单片段、单镜、背景、群体、泛称、只提及、无法确认同一具体人物，或生产上无需单人脸模连续。
- `production_task`、`state_task`、`template_task` 去向是否都能在 `global-task-draft` 找到对应 `task_id`。
- `manual_review_required` 是否只出现在任务行 `review_status` 或不可形成任务的去向记录中。
- `grouped_into` 是否指向真实模板任务、集合任务、别名目标或可见排除记录。

以上检查失败时，`severity` 必须为 `blocking`，并把 `affected_layer` 指向 `asset-registry.md`。

## 边界

- 不重新抽取资产。
- 不重新拆状态。
- 不重新评级。
- 不建立锚点。
- 不生成提示词。
- 不修改最终 Markdown 输出。

只使用以上列举过的字段和职责。
