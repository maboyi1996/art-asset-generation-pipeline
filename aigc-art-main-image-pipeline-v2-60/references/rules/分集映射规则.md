# 分集映射规则

## 规则目的

本文件只定义 `episode-usage-map.md` 如何生成分集使用映射表并引用全局资产任务。

分集映射接收前置层级已经确认的资产、状态、等级、候选去向和任务行，只负责回答：

> 每一集、每一场使用哪个资产状态，对应全局任务表中的哪个 `task_id` 或哪个可见排除/非视觉去向？

## 输入来源

分集映射必须基于以下前置结果生成：

- `scene-coverage-table`
- `global-asset-registry`
- `state-variant-result`
- `global-task-draft`
- `character-disposition-ledger`
- `generic-template-coverage-ledger`

资产存在、同一资产、状态拆分、生成任务均由前置层级决定；分集映射直接继承这些结果。

## 映射原则

- 分集映射行只引用全局任务表已有任务、人物候选去向台账中的可见排除/非视觉去向，或泛称模板覆盖台账中的覆盖记录。
- 同一资产状态在多个集数或场次出现时，分集映射指向同一个 `task_id`。
- 一个分集映射行可以覆盖同一集内多个 `scene_ids`。
- `priority_level`、`task_type`、`review_status` 必须继承全局任务表，不在分集映射中重新定义。
- 生产提示词只存在于全局生产任务表。
- 复用通过已有 `task_id` 表达。

## 分集使用行字段

每条分集映射行应包含：

| 字段 | 规则 |
|---|---|
| `episode_id` | 必填。 |
| `scene_ids` | 必填；同一集内多个场次可合并。 |
| `asset_type` | `character`、`scene`、`prop`。 |
| `asset_name` | 继承资产注册表的规范化名称。 |
| `state_name` | 继承状态拆分结果；无拆分时写默认状态。 |
| `priority_level` | 继承全局任务表；可见排除或非视觉去向可为空并说明。 |
| `task_id` | `review_status` 不是 `excluded` 且存在生产任务时必填。 |
| `task_type` | `review_status` 不是 `excluded` 且存在生产任务时继承全局任务表。 |
| `review_status` | 继承全局任务表或可见排除记录。 |
| `disposition_ref` | 可见排除、非视觉或泛称覆盖时填写对应 ledger ID。 |
| `episode_function` | 简短说明本集使用功能。 |
| `notes` | 只写复用、审核或排除说明。 |

分集映射字段只使用上表列举字段；新增字段必须先更新本规则。

## 复用映射

同一资产状态再次出现时：

- 使用已有 `task_id`。
- 在分集映射中记录新的 `episode_id` 和 `scene_ids`。
- 生产任务和提示词继续引用原全局任务行。

是否能复用同一资产或同一状态，由资产评级规则、状态拆分规则和全局任务表决定。

## 可见排除行

如果某项排除后可能让审核人员误以为遗漏，分集映射可以保留一条可见排除行。

可见排除行必须：

- 使用 `review_status: excluded`。
- 填写 `notes` 或引用对应 ledger 的排除原因。
- 填写 `disposition_ref`，指向人物候选去向、泛称模板覆盖或任务草稿中的可见排除记录。
- 只包含必要映射字段、审核状态和排除原因。

可见排除行只用于审核可见性，不计入生产任务数。

## 边界

- 不重新抽取资产。
- 不重新注册资产。
- 不重新拆状态。
- 不重新评级。
- 不补建任务。
- 不生成提示词。
- 不改变候选去向。
