# 角色锚定规则

## 规则目的

本文件只用于根据已完成的 `asset-rating-result`、`state-variant-result` 和 `global-task-draft` 建立人物脸模锚点关系。

角色锚定的核心问题是：

> 哪些人物任务需要固定脸模？哪些状态图必须引用已确认的人脸锚点？

本文件只定义人物锚点范围、锚点依赖、锁脸要求和 `character-anchor-result` 输出字段。

本文件不定义人物评级、状态拆分标准、任务类型枚举、分集映射、提示词模板或最终输出表格式。

## 输入依据

角色锚定必须读取前置结果：

- `global-task-draft`
- `asset-rating-result`
- `state-variant-result`

后续锚定行为必须继承 `asset-rating-result.priority_level`。不得在本文件重新评级。

## 锚定范围

| 人物等级 | 锚定行为 |
|---|---|
| `S` | 必须建立项目级人脸锚点。 |
| `A` | 必须建立项目级人脸锚点。 |
| `B` | 需要单集内脸模一致；默认不建立项目级锚点。B 级成立依据来自人物评级层的单集多片段风险判断。 |
| `C` | 不需要脸模锚点。 |

本表只解释不同等级在锚定层的处理方式，不用于判断人物等级。

## S/A 锚定流程

每个 `S/A` 人物必须：

1. 在 `global-task-draft` 中确认存在基础人物主图任务。
2. 将基础人物主图任务登记为项目级人脸锚点来源。
3. 分配稳定的 `anchor_task_id`。
4. 将同一人物后续可见状态图关联到该 `anchor_task_id`。
5. 标记锚点图确认状态：待确认、已确认、需人工确认。
6. 如果锚点图尚未确认，依赖状态行必须保留，并在锚定结果中标记人工确认原因。

如果 `S/A` 人物缺少基础人物主图任务，本层只输出锚定问题，不自行补建任务。

## B 级人物锚定范围

`B` 级人物需要本集内脸模一致，但默认不建立项目级人脸锚点。

本层不得重新判断一个人物是否为单片段、单镜或多片段风险人物；该判断只由 `人物评级规则.md` 在 `asset-rating-result` 中完成。

本层对 `B` 级人物只检查：

- 是否存在可被本集复用的人物基础任务。
- 是否有超出单集范围的锚定需求。
- 是否出现多状态图依赖锚点的异常情况。

如果 `B` 级人物需要多状态锚定工作流，本层只输出人工确认原因，不自行升级为 `A`，也不重写评级结果。

## C 级人物锚定范围

`C` 级人物不需要脸模锚点。

适用于：

- 单片段或单镜人物。
- 泛称龙套。
- 群体标签。
- 只被提及的人物。
- 同一人物证据不足的角色。
- 可选模板人物。

除非用户明确要求模板组，否则不得为 `C` 级人物创建依赖锚定图的状态行。

## `image_text_edit` 锁脸要求

人物状态图使用锚定图时必须：

- 保持锚定图中的同一个人。
- 保持脸型、五官比例、眼距、鼻子、嘴、下颌、肤色基调、年龄段、核心眼神。
- 只改变状态拆分结果支持的可见细节，例如服装、发型、妆容、伤病、疲惫、脏污、血迹、湿发、怀孕、阶层造型。
- 禁止美化、减龄、瘦脸、改变族群/地域特征、改变脸部比例、替换成另一张脸。

具体 `task_type` 取值合法性由 `任务类型字段审核规则.md` 约束。

## 输出字段

`character-anchor-result` 只使用以下字段：

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `base_asset_id` | 是 | 人物 base asset ID |
| `display_name` | 是 | 人物规范名 |
| `priority_level` | 是 | 继承 `asset-rating-result` |
| `base_task_id` | 条件必填 | `S/A/B` 已有基础人物任务 ID |
| `anchor_task_id` | `S/A` 必填 | 项目级人脸锚点任务 ID |
| `state_asset_id` | 状态行必填 | 需要引用锚点的状态资产 ID |
| `state_task_id` | 状态行必填 | 状态图任务 ID |
| `identity_reference_image` | 条件必填 | 锚点图状态或引用 |
| `identity_lock_status` | 是 | `pending/approved/manual_review_required/not_required` |
| `identity_lock_reason` | 是 | 锚定、无需锚定或需人工确认原因 |
| `lock_face_instruction` | 状态行必填 | 锁脸要求摘要 |

## 边界

- 不重新判断人物等级。
- 不拆分人物状态。
- 不创建分集映射。
- 不生成提示词正文。
- 不定义机器长表完整字段。

只使用以上列举过的字段和职责。
