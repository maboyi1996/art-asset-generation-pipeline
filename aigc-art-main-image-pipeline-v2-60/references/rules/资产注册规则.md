# 资产注册规则

## 规则目的

本文件只用于约束 `asset-registry.md` 如何把已抽取、已拆状态、已评级的候选资产注册为稳定资产 ID、状态 ID 和别名归并关系。

本规则解决的唯一问题是：

> 哪些候选可以合并为同一个 base asset 或 state asset，哪些别名必须保留为可审计映射？

本文件不生成任务、不判断人物去向、不记录污染清洗、不生成分集映射、不建立脸模锚点。

## 输入依据

必须读取前置层级已经产出的结构化结果：

- `asset-candidate-list`
- `state-variant-result`
- `asset-rating-result`

不得跳过前置结果重新抽取资产、重新拆状态或重新评级。

## 输出结果

输出结果固定命名为：

- `global-asset-registry`
- `alias-resolution-map`

## `global-asset-registry` 字段

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `base_asset_id` | 是 | 稳定资产 ID。 |
| `asset_type` | 是 | `character`、`scene`、`prop`。 |
| `asset_name` | 是 | 注册后的规范资产名，不得包含污染修饰。 |
| `source_candidate_ids` | 是 | 对应候选 ID 列表。 |
| `priority_level` | 是 | 继承 `asset-rating-result`，不得重新评级。 |
| `same_asset_evidence` | 是 | 合并为同一资产的证据强度和说明。 |
| `state_asset_ids` | 条件必填 | 有状态拆分时填写状态 ID 列表。 |
| `episode_ids` | 是 | 来源集列表。 |
| `scene_ids` | 是 | 来源场次列表。 |
| `registry_status` | 是 | `registered`、`alias_only`、`excluded_from_registry`、`manual_review_required`。 |
| `registry_note` | 条件必填 | 证据不足、冲突、不合并或送审时填写。 |

## `alias-resolution-map` 字段

| 字段 | 是否必填 | 规则 |
|---|---:|---|
| `alias_id` | 是 | 稳定别名记录 ID。 |
| `raw_name` | 是 | 原文名称或称呼。 |
| `normalized_name` | 是 | 规范名。 |
| `target_base_asset_id` | 条件必填 | 已合并到注册资产时填写。 |
| `source_candidate_id` | 是 | 来源候选 ID。 |
| `alias_type` | 是 | `same_person_alias`、`title_alias`、`location_alias`、`prop_alias`、`unresolved_alias`。 |
| `resolution_status` | 是 | `resolved`、`not_merged`、`manual_review_required`。 |
| `resolution_evidence` | 是 | 支持归并或不归并的证据。 |
| `uncertainty_note` | 条件必填 | 可转移称谓、同名不同人、泛称或证据冲突时填写。 |

## 合并原则

- 同一具体人物、场景或道具证据不足时，默认不合并。
- 泛称人物、泛化场景或普通道具不得只因名称重复自动合并。
- 可转移称谓必须绑定具体人物和时间范围；无法消歧时写入 `alias-resolution-map` 的 `manual_review_required` 或 `not_merged`。
- 状态 ID 必须来自 `state-variant-result`，不得在注册层重新拆状态。
- `priority_level` 必须继承 `asset-rating-result`，不得在注册层重新评级。

## 审计交接要求

- `global-asset-registry` 中的资产名不得包含动作、情绪、语气、声音、瞬时表情、组合词或未消歧可转移称谓。
- 每个已注册资产必须能回溯到 `asset-candidate-list`。
- 每个状态 ID 必须能回溯到 `state-variant-result`。
- 每个别名合并必须有证据；证据不足时必须保留不合并或人工复核记录。
- 污染标签不得通过别名映射绕过准入门。

## 边界

- 不重新抽取资产。
- 不重新拆状态。
- 不重新评级。
- 不生成候选去向。
- 不生成任务。
- 不生成分集映射。
- 不建立脸模锚点。
- 不生成提示词。
- 不定义最终 Markdown 排版。
