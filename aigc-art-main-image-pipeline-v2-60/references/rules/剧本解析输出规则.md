# 剧本解析输出规则

## 规则目的

本文件约束 `script-intake-result` 的生成方式。

本层的目的不是识别资产，而是把原始剧本稳定拆成“项目、集、场、原文文本块、来源定位”五类基础信息，确保后续每一个人物、场景、道具、状态、评级和任务都能回溯到原始剧本文本。

## 输入

- 原始剧本、PDF、小说稿、分集文本或用户提供的等价文本材料。
- 用户额外说明的项目名、集数范围、版本信息或特殊解析要求。

## 输出

输出结果固定命名为 `script-intake-result`。

`script-intake-result` 是后续 `scene-coverage-table` 的唯一解析依据。后续层级不得重新发明集号、场号或文本范围。

## 必填字段

| 字段 | 规则 |
|---|---|
| `project_title` | 项目名。剧本内无法确定时，使用文件名或用户给定名称。 |
| `source_file` | 原始文件路径、文件名或用户提供文本来源。 |
| `source_version` | 文件版本、完稿版、修订版、日期等可识别版本信息；无法确定时写 `unknown`。 |
| `episode_list` | 已识别集数列表，使用稳定 `episode_id`。 |
| `scene_index` | 每场的稳定索引，至少包含 `episode_id`、`scene_id`、原场次标题、内部场次标题、文本范围。 |
| `source_locator` | 页码、段落、行号、章节位置或可回溯文本块；必须足够支持后续核查。 |
| `parse_notes` | 缺页、断场、标题异常、重复场次、集数不明、跨页断句等解析问题。 |

## `scene_index` 字段要求

| 字段 | 规则 |
|---|---|
| `episode_id` | 使用稳定编号，例如 `E01`、`E02`。无法识别真实集号时，按文本顺序生成临时编号并在 `parse_notes` 说明。 |
| `scene_id` | 在单集内稳定唯一，例如 `E01-S001`。不要依赖原文中可能重复或缺失的场号。 |
| `raw_scene_heading` | 原文场次标题，保持原样。 |
| `normalized_scene_heading` | 便于后续引用的内部标题，可根据地点、时间、事件简写。 |
| `text_range` | 该场对应的文本范围或文本块编号。 |
| `source_locator` | 该场对应页码、段落、行号或原文片段定位。 |

## 场次标题识别

场次标题必须同时支持以下常见写法：

- `65-2 宋府 日 内`
- `65-2（闪回）宋府 日 内`
- `65-2(闪回) 宋府 日 内`
- `65 - 2 宋府 日 内`

识别场次标题时，不得要求场号和后续标题之间必须有空格。场号后的括号标记，例如“闪回”“回忆”“梦境”“传闻画面”，应保留在 `raw_scene_heading`，并在 `normalized_scene_heading` 或 `parse_notes` 中标记叙事形态。

如果一行文本符合“集号-场号”标题形态，却被保留在上一场 `text_range` 内，必须在 `parse_notes` 标记 `possible_unparsed_scene_heading`，并优先回到本层修正场次边界。后续层级不得用重新切场来掩盖解析错误。

## 解析原则

- 以“可稳定回溯”为第一目标，不以“漂亮摘要”为目标。
- 原文场次标题不稳定时，保留原文标题，同时创建稳定内部 ID。
- 场次边界不确定时，不跳过文本块；应保留文本，并在 `parse_notes` 标记边界疑点。
- 如果剧本没有明确分场，应按可见的叙事段落、地点变化、时间变化或行动段落建立临时场次。
- 如果剧本没有明确分集，应按用户给定范围或文本自然分段建立临时 `episode_id`，并说明依据。

## 边界

本层只做文本解析和来源定位：

- 不抽取人物、场景、道具。
- 不合并别名。
- 不判断两个称呼是否为同一资产。
- 不拆分状态。
- 不做 S/A/B/C 评级。
- 不判断任务类型。
- 不生成提示词。

只使用以上列举过的字段和职责。

## 异常处理

| 异常 | 处理方式 |
|---|---|
| 集号缺失 | 生成临时 `episode_id`，在 `parse_notes` 写明依据。 |
| 场号缺失或重复 | 生成稳定 `scene_id`，保留原文标题。 |
| 场号后无空格或带括号标记 | 仍识别为新场，保留原文标题，在 `parse_notes` 标记标题格式。 |
| 疑似场次标题残留在上一场文本中 | 标记 `possible_unparsed_scene_heading`，该结果不得直接进入后续资产生产。 |
| 场次边界不清 | 不丢弃文本，建立临时场次并标记 `boundary_uncertain`。 |
| PDF 页码与文本顺序不一致 | 优先保持文本阅读顺序，同时记录页码异常。 |
| 文件疑似缺页或断页 | 在 `parse_notes` 标记，不自行补写剧情。 |

## 示例

| 原文情况 | 正确处理 |
|---|---|
| “第一集”下有多个无编号场景 | 建立 `E01-S001`、`E01-S002`，原文标题缺失时补内部标题。 |
| 场次标题写作“宋府/夜”，后文跨页继续 | 同一场保留为一个 `scene_id`，`source_locator` 记录跨页范围。 |
| `65-2（闪回）宋府 日 内` | 识别为新场，保留原文标题，并标记为闪回可见场次。 |
| 文中出现“棠宁冷笑” | 本层不记录人物资产，只保留在该场文本块中，交给下一层覆盖记录。 |
